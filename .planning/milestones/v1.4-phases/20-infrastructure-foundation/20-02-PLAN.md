  </read_first>
  <action>
    Per D-04, D-05, D-06, D-07, D-10:

    1. Add a `redis` service block between the `mongo` and `api` services in docker-compose.yaml:
       ```yaml
       # Redis Cache
       redis:
         container_name: trade-flow-redis
         image: redis:7.4-alpine
         ports:
           - "6379:6379"
         command: redis-server --maxmemory-policy noeviction
         networks:
           - app-network
         healthcheck:
           test: ["CMD", "redis-cli", "ping"]
           interval: 10s
           timeout: 5s
           retries: 5
       ```

       Key details:
       - `redis:7.4-alpine` image (smaller alpine variant, sufficient for dev)
       - `command:` override sets maxmemory-policy noeviction (D-06 -- BullMQ silently loses jobs otherwise)
       - No volume mount (D-05 -- ephemeral storage, queue jobs are transient)
       - Health check uses `redis-cli ping` (D-07 -- matches existing mongo health check pattern)
       - On `app-network` bridge (D-04 -- matches existing pattern)
       - Container name `trade-flow-redis` (follows convention: trade-flow-{service})

    2. Update the `api` service `depends_on` section to add Redis alongside existing MongoDB:
       ```yaml
       depends_on:
         mongo:
           condition: service_healthy
         redis:
           condition: service_healthy
       ```

    3. Add `REDIS_URL` to the `api` service `environment` section (per D-10):
       ```yaml
       REDIS_URL: redis://redis:6379
       ```
       IMPORTANT: Uses Docker service name `redis` (not `localhost`) for container-to-container networking on the bridge network.

    Do NOT add a volume for Redis (D-05: ephemeral storage).
    Do NOT create a redis.conf file (command override is sufficient for one flag).
