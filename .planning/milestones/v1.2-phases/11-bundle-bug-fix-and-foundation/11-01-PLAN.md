Component API:
```typescript
interface SearchableItemPickerProps {
  items: Item[];                    // Available items to display
  excludeItemIds?: string[];        // Item IDs to hide (already selected)
  onSelect: (item: Item) => void;   // Called when user picks an item
  placeholder?: string;             // Trigger button text (default: "Select item...")
  disabled?: boolean;
}
```

Implementation following CreateJobDialog Popover+Command pattern:
1. State: `popoverOpen` (boolean), `search` (string)
2. Filter items: useMemo that filters out `excludeItemIds` and filters by search term (case-insensitive name match). Exclude bundle-type items and inactive items.
3. Group filtered items: useMemo that groups by `item.type` into `{ material: Item[], labour: Item[], fee: Item[] }`
4. Render Popover+Command with `shouldFilter={false}`:
   - PopoverTrigger: Button with `variant="outline"`, `role="combobox"`, shows placeholder text + ChevronsUpDown icon
   - CommandInput: placeholder "Search items...", controlled by `search` state
   - CommandEmpty: "No items found."
   - Three CommandGroup sections with headings "Materials", "Labour", "Fees" -- only render groups that have items
   - Each CommandItem shows:
     - Item name (left)
     - Badge with item type (Material/Labour/Fee) using `variant="secondary"` and appropriate text
     - Formatted default price using `useBusinessCurrency().formatAmount()` -- show "No price" if `defaultPrice` is null
   - On select: call `onSelect(item)`, set `popoverOpen(false)`, set `search("")`

Use `@/` import alias. Import types with `import type`. Use `cn()` for conditional classes.
