<task type="auto">
  <name>Task 1: Fix Money.percentageOf math bug</name>
  <files>trade-flow-api/src/core/value-objects/money.value-object.ts</files>
  <action>
Fix the `percentageOf(base: Money): number` method in Money value object. The current implementation divides a Dinero object by `base.toMinorUnits()` (an integer like 10000 for $100.00), which causes integer division in Dinero to return 0 for any value smaller than the base.

Replace the body with pure numeric calculation:
```typescript
public percentageOf(base: Money): number {
  if (base.isZero()) {
    return 0;
  }
  return (this.toMinorUnits() / base.toMinorUnits()) * 100;
}
```

This avoids Dinero's integer division entirely and computes the percentage as a plain number. For example, if `this` = $15 (1500 minor) and `base` = $100 (10000 minor), result = (1500/10000)*100 = 15.

Do NOT change any other method in Money. The bug is isolated to percentageOf.
