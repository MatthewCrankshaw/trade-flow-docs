## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-49-01 | Tampering | reviseEstimate mutation | accept | Backend EstimatePolicy enforces ownership via JWT. Frontend is not a trust boundary -- server validates all revisions. No body sent (POST is clone-only). |
| T-49-02 | Information Disclosure | getEstimateRevisions query | accept | Backend EstimatePolicy enforces canRead on all revisions. Only the authenticated estimate owner can retrieve history. |
| T-49-03 | Spoofing | estimateId in URL/mutation | accept | Firebase JWT authentication via existing prepareHeaders in apiSlice. Backend verifies token on every request. |
| T-49-04 | Denial of Service | Rapid revise clicks | mitigate | Button disabled during mutation via `isRevising` state -- prevents double-submit. Backend idempotency is out of scope (no body = new revision each time, which is correct behavior). |
</threat_model>
