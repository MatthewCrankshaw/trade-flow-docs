2. **Redis AOF startup check in `src/worker.ts` with conditional processor registration (D-AOF-01):** Add an AOF check after the `NestFactory.createApplicationContext(WorkerModule)` call. Per D-AOF-01, if AOF is not enabled, log a fatal error AND refuse to register the `ESTIMATE_FOLLOWUPS` processor. Other processors (STRIPE_WEBHOOKS, ECHO) continue running.

   Read the existing `src/worker.ts` carefully — it likely calls `app.listen()` or registers workers after context creation. The approach is:

   ```typescript
   import IORedis from "ioredis";
   import { AppLogger } from "@core/services/app-logger.service";
   import { ConfigService } from "@nestjs/config";

   // After app context creation:
   const logger = new AppLogger("WorkerBootstrap");
   const configService = app.get(ConfigService);
   const redisUrl = configService.get<string>("REDIS_URL");
   const redisClient = new IORedis(redisUrl);

   let aofEnabled = false;
   try {
     const result = await redisClient.config("GET", "appendonly");
     // result format: ["appendonly", "yes"] or ["appendonly", "no"]
     aofEnabled = Array.isArray(result) && result[1] === "yes";
     if (!aofEnabled) {
       logger.error(
         "FATAL: Redis appendonly is NOT enabled. Refusing to register ESTIMATE_FOLLOWUPS processor. " +
         "Scheduled follow-up jobs will not be processed. " +
         "Enable AOF persistence: redis-server --appendonly yes --appendfsync everysec"
       );
     } else {
       logger.log("Redis AOF persistence verified: appendonly=yes");
     }
   } catch (error) {
     logger.error("Failed to check Redis AOF config — ESTIMATE_FOLLOWUPS processor will not be registered", (error as Error).stack);
     // aofEnabled remains false, processor will be skipped
   } finally {
     await redisClient.quit();
   }
   ```

   // Always register STRIPE_WEBHOOKS, ECHO, etc.

   **Read `src/worker.ts` and `src/worker/worker.module.ts` before deciding.** Choose whichever approach cleanly refuses processor registration on AOF failure. The acceptance criterion is: if Redis AOF is off, `ESTIMATE_FOLLOWUPS` jobs are NOT processed (they remain queued or move to failed, not silently dropped).
