       - Apply muted styling for past dates using the `modifiers` and `modifiersClassNames` props: `modifiers={{ past: { before: new Date() } }}` with `modifiersClassNames={{ past: "text-muted-foreground" }}`
       - Past dates ARE selectable (per CONTEXT.md: for logging visits that already happened)

       - Generate time options: 15-minute intervals from 6:00 AM (06:00) to 9:00 PM (21:00) in 12-hour format labels
       - Use `useController` for `startTime` field
       - Render shadcn Select with `<SelectValue placeholder="Select time" />`

    3. **Duration** -- Select dropdown:
       - Fixed presets: `[{ value: 30, label: "30 minutes" }, { value: 60, label: "1 hour" }, { value: 90, label: "1.5 hours" }, { value: 120, label: "2 hours" }, { value: 180, label: "3 hours" }, { value: 240, label: "4 hours" }, { value: 600, label: "Full day" }]`
       - Use `useController` for `durationMinutes` field
       - Note: Select values are strings, so convert: `onValueChange={(v) => field.onChange(Number(v))}` and `value={String(field.value)}`

    4. **Visit Type** -- color-coded Select dropdown (per CONTEXT.md):
       - Placeholder: "Select visit type (optional)"
       - Each SelectItem shows color dot (3x3 rounded-full span with backgroundColor from vt.color) + name
       - Disabled when no active visit types exist
       - When disabled, show helper text: "No active visit types -- create one in Settings"
       - Use `useController` for `visitTypeId` field

    5. **Notes** -- Textarea:
       - Always visible (not collapsed), per CONTEXT.md
       - Placeholder: "Add notes for this visit (optional)"
       - Use `useController` for `notes` field
       - `className="min-h-20 resize-none"` matching VisitTypeFormDialog textarea

    - Compose ISO8601 startDateTime from date + startTime: `${format(values.date, 'yyyy-MM-dd')}T${values.startTime}:00.000Z` (using date-fns format)
    - Build CreateScheduleRequest: always include startDateTime and durationMinutes; only include visitTypeId if non-empty string; only include notes if non-empty trimmed string
    - Call `createSchedule(request)`
    - On success: `toast.success("Visit scheduled", { description: "Visit scheduled for ${format(values.date, 'EEE d MMM yyyy')}" })`, then `onClose()`
