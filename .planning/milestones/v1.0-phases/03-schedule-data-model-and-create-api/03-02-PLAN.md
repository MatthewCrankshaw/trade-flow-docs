    - ScheduleCreatorService.create() skips visit type validation when visitTypeId is null
    - ScheduleCreatorService.create() throws InvalidRequestError with SCHEDULE_JOB_NOT_FOUND when job retriever throws
    - ScheduleCreatorService.create() throws InvalidRequestError with SCHEDULE_VISIT_TYPE_NOT_FOUND when visit type retriever throws
    - ScheduleCreatorService.create() delegates to AuthorizedCreatorFactory.createFor() with scheduleRepository and schedulePolicy
    - ScheduleCreatorService.create() returns the created schedule DTO

    **ScheduleCreatorService (`services/schedule-creator.service.ts`):**
    - Clone from `JobCreatorService` pattern
    - `@Injectable()`, implements `ICreatorService`
    - Create `AppLogger` instance: `new AppLogger(ScheduleCreatorService.name)`
    - Constructor injects:
      - `ScheduleRepository`
      - `SchedulePolicy`
      - `AuthorizedCreatorFactory`
      - `JobRetrieverService` (from `@job/services/job-retriever.service`)
      - `VisitTypeRetrieverService` (from `@visit-type/services/visit-type-retriever.service`)
    - Method `create(authUser: IUserDto, schedule: IScheduleDto): Promise<IScheduleDto>`:
      1. **Validate job exists and user has access:**
         ```
         try {
           await this.jobRetriever.findByIdOrFail(authUser, schedule.jobId);
         } catch (error) {
           this.logger.warn("Job not found for schedule creation", { jobId: schedule.jobId });
           throw new InvalidRequestError(ErrorCodes.SCHEDULE_JOB_NOT_FOUND, `Job with id ${schedule.jobId} not found for schedule creation`);
         }
         ```
      2. **Validate visit type if provided:**
         ```
         if (schedule.visitTypeId) {
           try {
             await this.visitTypeRetriever.findByIdOrFail(authUser, schedule.visitTypeId);
           } catch (error) {
             this.logger.warn("Visit type not found for schedule creation", { visitTypeId: schedule.visitTypeId });
             throw new InvalidRequestError(ErrorCodes.SCHEDULE_VISIT_TYPE_NOT_FOUND, `Visit type with id ${schedule.visitTypeId} not found for schedule creation`);
           }
         }
         ```
      3. **Create via authorized creator:**
         ```
         const authorizedCreator = this.authorizedCreatorFactory.createFor(this.scheduleRepository, this.schedulePolicy);
         return authorizedCreator.create(authUser, schedule);
         ```
    - NOTE on assignee validation: For the solo-operator MVP, the assignee defaults to auth user (SCHED-06) and the SchedulePolicy already validates business membership of the auth user. Full assignee membership validation for other users is deferred to FUT-04 (team support). Add a `// TODO: Validate assignee business membership when team support is added (FUT-04)` comment.

    **ScheduleController (`controllers/schedule.controller.ts`):**
    - Clone from `VisitTypeController` create method pattern
    - `@Controller("v1")`, create `AppLogger` instance
    - Constructor injects `ScheduleCreatorService` only (no retriever/updater -- Phase 3 scope is create only)
    - Single endpoint:
      ```
      @UseGuards(JwtAuthGuard)
      @Post("business/:businessId/job/:jobId/schedule")
      public async create(
        @Req() request: { user: IUserDto; params: { businessId: string; jobId: string } },
        @Body() requestBody: CreateScheduleRequest,
      ): Promise<IResponse<IScheduleResponse>> {
        try {
          const { businessId, jobId } = request.params;
          const schedule = mapCreateScheduleRequestToDto(requestBody, jobId, businessId, request.user.id);
          const createdSchedule = await this.scheduleCreator.create(request.user, schedule);
          const response = mapScheduleToResponse(createdSchedule);
          return createResponse([response]);
        } catch (error) {
          const httpError = createHttpError(error);
          throw httpError;
        }
      }
      ```
    - NOTE: The URL nests under both business and job: `business/:businessId/job/:jobId/schedule`. This scopes schedule creation to a specific job, and `businessId` + `jobId` are both extracted from params.
