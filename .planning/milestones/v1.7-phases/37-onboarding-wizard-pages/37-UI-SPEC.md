## Color

All values from the established Trade Flow colour system (consistent with Phase 36):

| Role | Token | OKLCH | Usage in Phase 37 |
|------|-------|-------|-------------------|
| Dominant (60%) | `background` | `oklch(0.99 0 0)` | Wizard page background (with subtle gradient overlay) |
| Secondary (30%) | `card` | `oklch(1 0 0)` | Wizard card surface, trade card surfaces |
| Accent (10%) | `primary` | `oklch(0.31 0.09 245)` | See reserved list below |
| Destructive | `destructive` | `oklch(0.64 0.21 25)` | Trial badge at <=3 days remaining |

Accent (`primary`) reserved for:
- "Continue Setup" CTA button on Step 1 (`bg-primary text-primary-foreground`)
- "Set up my business" CTA button on Step 2 (`bg-primary text-primary-foreground`)
- Progress bar filled segment (`bg-primary`)
- Selected trade card border highlight (`border-primary`)

Additional semantic colors used:
- `warning` (existing token): Trial badge at <=10 days remaining (`text-yellow-600` / `bg-yellow-100`)
- `destructive` (existing token): Trial badge at <=3 days remaining (`text-destructive` / `bg-destructive/10`)
- `muted-foreground`: Unselected trade card text, step subtitles, progress text

## Interaction States

### Step 1: Profile Setup

| State | Condition | Visual |
|-------|-----------|--------|
| Default | User lands on /onboarding, no display name | Card with "What's your name?" heading, empty display name input, disabled "Continue Setup" button |
| Input valid | Display name field has 1+ non-whitespace characters | "Continue Setup" button becomes enabled (`bg-primary`) |
| Submitting | User clicks "Continue Setup" | Button shows `Loader2` spinner, input disabled, text changes to "Saving..." |
| Success | PATCH /v1/user returns 200 | Instant swap to Step 2 content (no animation per D-06) |
| Error | API call fails | Sonner error toast: "Something went wrong. Please try again." Button re-enables. |

### Step 2: Business Setup

| State | Condition | Visual |
|-------|-----------|--------|
| Default | Step 2 renders | Card with "Tell us about your business" heading, empty business name input, trade card grid with no selection, disabled "Set up my business" button |
| Partial (name only) | Business name entered, no trade selected | Button remains disabled |
| Partial (trade only) | Trade selected, no business name | Button remains disabled |
| Trade selected | User clicks a trade card | Card gets `border-primary ring-2 ring-primary/20` highlight, previous selection deselected |
| Other selected | User clicks "Other" trade card | "Other" card highlights, text input appears below grid with placeholder "e.g. Roofer, Glazier" |
| Form valid | Business name has 1+ non-whitespace chars AND trade selected (and custom name if "Other") | "Set up my business" button becomes enabled (`bg-primary`) |
| Submitting | User clicks "Set up my business" | Transitions to SetupLoadingScreen (replaces wizard card) |

### Setup Loading Screen

| State | Condition | Visual |
|-------|-----------|--------|
| Loading | POST /v1/business in progress | Full-screen centered: `Loader2` (h-10 w-10 animate-spin text-primary) + "Setting up your business..." text below |
| Business created, trial starting | POST /v1/business succeeded, POST /v1/subscription/trial in progress | Same loading screen, text changes to "Starting your free trial..." |
| Success | Both API calls complete | Sonner success toast: "You're all set! Your 30-day free trial has started." Navigate to `/dashboard` |
| Error | Either API call fails | Loading spinner replaced by `AlertCircle` icon (h-10 w-10 text-destructive) + error message + "Try again" button |

### Trial Badge (Header)

| State | Condition | Visual |
|-------|-----------|--------|
| Normal (>10 days) | trialDaysRemaining > 10 | Desktop: `Badge variant="secondary"` with `Clock` icon + "28 days left". Mobile: `Badge variant="secondary"` with "28d" only |
| Warning (4-10 days) | trialDaysRemaining >= 4 AND <= 10 | Desktop: `Badge` with yellow background (`bg-yellow-100 text-yellow-800`) + `Clock` icon + "7 days left". Mobile: same yellow styling + "7d" |
| Urgent (1-3 days) | trialDaysRemaining >= 1 AND <= 3 | Desktop: `Badge` with destructive styling (`bg-destructive/10 text-destructive`) + `Clock` icon + "2 days left". Mobile: same destructive styling + "2d" |
| Last day | trialDaysRemaining === 0 | Desktop: "Trial ends today". Mobile: "0d". Destructive styling. |
| Hidden | Subscription status is not "trialing" | Badge not rendered |
| Click | User clicks badge | Opens Stripe Billing Portal in new tab (calls manage endpoint for portal URL) |

### OnboardingGuard

| State | Condition | Visual |
|-------|-----------|--------|
| Loading | User/business data not yet fetched | Centered `Loader2` spinner (same as Phase 36 suspense fallback) |
| Needs onboarding | No display name OR no business | `Navigate to="/onboarding"` redirect |
| Resume Step 2 | Has display name but no business | `Navigate to="/onboarding"` -- wizard detects display name exists, renders Step 2 |
| Complete | Has display name AND business AND subscription | Renders `<Outlet />` (pass-through) |

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| Mobile (< 640px) | Wizard card full-width with `px-4` page padding. Trade card grid 2 columns (`grid-cols-2`). Trial badge shows compact "{N}d" format. |
| Tablet (>= 640px) | Trade card grid 3 columns (`sm:grid-cols-3`). Wizard card still full-width within max-width constraint. |
| Desktop (>= 768px) | Wizard card `max-w-[480px]` centered. Trial badge shows full "{N} days left" format with `Clock` icon. |

## Accessibility Notes

- Wizard card heading uses `<h1>` on each step (single h1 per page)
- All form inputs have associated `<Label>` elements with `htmlFor` attribute
- Trade cards are `<button>` elements with `aria-pressed` state for selection
- Selected trade card has visual focus ring (`ring-2 ring-primary/20`) meeting 3:1 contrast ratio
- Progress bar has `role="progressbar"` with `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"`
- Loading spinner has `aria-label="Loading"` or visually hidden text
- Error state uses `role="alert"` for screen reader announcement
- Trial badge is a `<button>` with `aria-label="Trial: {N} days remaining. Click to manage subscription"`
- Form validation errors announced via `aria-describedby` on input fields
- Keyboard: Tab navigates between trade cards, Enter/Space selects
