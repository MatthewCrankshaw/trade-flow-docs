    - Invalid operator (e.g., "filter:status:xx") is ignored (silently skipped)
    - Malformed filter key (e.g., "filter:status" with no operator, "filter:" empty) is ignored
    - Multiple filter params in same query are all parsed

    - Empty value string (e.g., "filter:status:in": "") is ignored

  <action>
    **Step 1: Create the interface file** `src/core/filters/structured-filter.interface.ts`:
    ```typescript
    export type FilterOperator = "eq" | "nq" | "lt" | "gt" | "bt" | "in";

    export interface IStructuredFilter {
      /** The field being filtered. Dot-separated for nested fields. */
      field: string;
      /** The 2-character operator. */
      operator: FilterOperator;
      /** The raw string value from the query parameter. */
      value: string;
    }
    ```

    **Step 2: Create the operators constant** `src/core/filters/filter-operators.constant.ts`:
    ```typescript
    import { FilterOperator } from "@core/filters/structured-filter.interface";

    export const FILTER_OPERATORS: ReadonlySet<string> = new Set<FilterOperator>([
      "eq", "nq", "lt", "gt", "bt", "in",
    ]);
    ```

    **Step 3: Create the parser** `src/core/filters/parse-structured-filters.utility.ts`:
    - Accept `query: Record<string, string | string[] | undefined>`
    - Iterate over query keys
    - For each key starting with `filter:`, split on `:` to get parts: `["filter", field, operator]`
    - The field portion is everything between the first and last colon (supports dot-separated nested fields)
    - Validate: must have exactly the pattern `filter:<field>:<op>` where `<op>` is in FILTER_OPERATORS
    - Parse strategy: split the key on `:` — first part must be "filter", last part (2 chars) is operator, everything in between joined by `:` is the field. BUT since field uses `.` separation (not `:`), a simpler approach: split key on `:` into exactly 3 parts: `["filter", "<field>", "<op>"]`. If more than 3 parts, skip (malformed).
    - Normalize value: if array, take first element. If undefined or empty string, skip.
    - Return `IStructuredFilter[]`
