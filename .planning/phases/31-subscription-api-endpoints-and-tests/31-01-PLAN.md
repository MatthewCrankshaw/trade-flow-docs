<objective>
Implement three subscription management endpoints (GET, DELETE, POST portal) and a global SubscriptionGuard that enforces subscription status on write operations.

Purpose: Users need to view their subscription status (BILL-01), cancel their subscription gracefully at period end (BILL-02), and access the Stripe Billing Portal for payment management (BILL-03). The guard (D-09 through D-13) prevents unsubscribed users from performing write operations at the API level, complementing the frontend SubscriptionGate.

Output: SubscriptionRetriever service, cancel and portal service methods, three new controller routes, SubscriptionGuard with @SkipSubscriptionCheck decorator, all registered in module and app.
</objective>
