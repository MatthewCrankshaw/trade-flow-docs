       ```
       id, businessId, customerId, jobId, number, title, notes?, status: QuoteStatus,
       quoteDate: string, validUntil?: string, sentAt?: string, acceptedAt?: string, rejectedAt?: string,
       ```

     - Follow jobApi.ts pattern exactly: `apiSlice.injectEndpoints({ endpoints: (builder) => ({ ... }) })`.
     - Endpoints:
       a. `getQuotes: builder.query<Quote[], string>` -- query: `/v1/business/${businessId}/quotes`, transformResponse extracts `response.data`, providesTags with per-item + LIST tags.
       b. `getQuote: builder.query<Quote, string>` -- query: `/v1/quote/${quoteId}`, transformResponse extracts `response.data[0]`, providesTags with id tag.
       c. `createQuote: builder.mutation<Quote, { businessId: string; data: CreateQuoteRequest }>` -- POST to `/v1/business/${businessId}/quote`, invalidatesTags LIST + Job LIST (since quotes appear on job detail).
       d. `transitionQuote: builder.mutation<Quote, { businessId: string; quoteId: string; status: QuoteStatus }>` -- POST to `/v1/business/${businessId}/quote/${quoteId}/transition`, body `{ status }`, invalidatesTags: quote id + LIST.
     - Export all hooks: `useGetQuotesQuery`, `useGetQuoteQuery`, `useCreateQuoteMutation`, `useTransitionQuoteMutation`.

  4. **Quotes feature barrel** (trade-flow-ui/src/features/quotes/index.ts):
     - Add export for api: `export * from "./api/quoteApi"`.
     - Keep existing component exports.

  Import types using `import type` as per project convention.
