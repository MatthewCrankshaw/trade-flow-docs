  <action>
Create `src/core/interfaces/query-filter.interface.ts` with:
```typescript
export type FilterOperator = "in" | "eq" | "nq" | "lt" | "gt" | "le" | "ge" | "bt";
export interface IQueryFilter {
  field: string;
  operator: FilterOperator;
  value: string | string[];
}
```

Create `src/core/utilities/query-filter-parser.utility.ts` following the `paginationQueryToBaseQueryOptions` analog pattern. Single exported function `parseQueryFilters(query: Record<string, string>): IQueryFilter[]`. Logic:
1. Iterate query entries, skip keys not starting with `"filter:"`
2. Strip `"filter:"` prefix, find last colon index to split field from operator
3. Validate operator against `VALID_OPERATORS` allowlist: `["in", "eq", "nq", "lt", "gt", "le", "ge", "bt"]` (per D-01)
4. Skip if field is empty or value is empty
5. For `"in"` and `"bt"` operators, split value on comma into string array
6. Push `{ field, operator, value }` to results
7. Also export a `escapeRegex(input: string): string` function that escapes regex special characters (`.*+?^${}()|[]\`) for safe use in MongoDB `$regex` queries. This prevents regex injection (T-54-01).

Create `src/core/utilities/query-sort-parser.utility.ts`. Single exported function `parseQuerySort(sort: string | undefined): Record<string, 1 | -1>`. Logic:
1. If sort is undefined or empty string, return `{}`
2. If sort starts with `"-"`, return `{ [sort.slice(1)]: -1 }`
3. Otherwise return `{ [sort]: 1 }`

Write tests FIRST in `src/core/test/utilities/query-filter-parser.utility.spec.ts` and `src/core/test/utilities/query-sort-parser.utility.spec.ts` covering all behavior cases above. Use the Jest test pattern from the codebase (describe/it blocks, no NestJS TestingModule needed -- these are pure functions).
