<objective>
Thread the user's createdAt timestamp from the MongoDB users collection through the support user aggregation pipeline, DTO, and controller response so the UI's Created column displays actual dates.

Purpose: Close UAT Gap 1 -- the Created column in UserListTable always shows "---" because createdAt is hardcoded to null in the controller. The users collection already stores createdAt; it just needs to be projected through all layers.

Output: GET /v1/support/users returns each user's createdAt as an ISO string (or null if missing).
</objective>
