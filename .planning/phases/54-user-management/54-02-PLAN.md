Follow `UserRepository` pattern. `@Injectable()` class with `private readonly logger = new AppLogger(SupportUserRepository.name)` and `private static readonly COLLECTION = "users"`.

Constructor injects `MongoDbFetcher`.

Method `findPaginated(options: IBaseQueryOptionsDto): Promise<{ data: ISupportUserDto[]; total: number }>`:
1. Build `$match` stage from options:
   - If `options.search`: add `$or: [{ name: { $regex: escapedSearch, $options: "i" } }, { email: { $regex: escapedSearch, $options: "i" } }]` using `escapeRegex` from `@core/utilities/query-filter-parser.utility`
   - If filter with field `"role.type"` and operator `"eq"`: 
     - value `"support"` -> add `{ supportRoleIds: { $exists: true, $ne: [] } }`
     - value `"customer"` -> add `{ $or: [{ supportRoleIds: { $exists: false } }, { supportRoleIds: { $size: 0 } }] }`
2. Build aggregation pipeline:
   - Stage 1: `$match` (from above, or `{}` if no filters)
   - Stage 2: `$lookup` subscription data using pipeline-style to handle ObjectId-to-string mismatch:
     ```
     {
       $lookup: {
         from: "subscriptions",
         let: { odUserId: { $toString: "$_id" } },
         pipeline: [
           { $match: { $expr: { $eq: ["$userId", "$$odUserId"] } } },
           { $project: { status: 1, trialEnd: 1, currentPeriodEnd: 1, canceledAt: 1, createdAt: 1 } },
         ],
         as: "subscriptionDocs",
       },
     }
     ```
   - Stage 3: `$unwind: { path: "$subscriptionDocs", preserveNullAndEmptyArrays: true }`
   - Stage 4: `$lookup` business data:
     ```
     {
       $lookup: {
         from: "businesses",
         let: { userBizIds: { $ifNull: ["$businessIds", []] } },
         pipeline: [
           { $match: { $expr: { $in: ["$_id", "$$userBizIds"] } } },
           { $project: { name: 1, trade: 1 } },
           { $limit: 1 },
         ],
         as: "businessDocs",
       },
     }
     ```
   - Stage 5: `$unwind: { path: "$businessDocs", preserveNullAndEmptyArrays: true }`
   - Stage 6: If subscription status filter exists (field `"subscription.status"`, operator `"eq"`), add post-lookup `$match: { "subscriptionDocs.status": filterValue }`
   - Stage 7: `$facet` for pagination:
     ```
     {
       data: [
         { $sort: sortStage || { _id: -1 } },
         { $skip: (page - 1) * limit },
         { $limit: limit },
       ],
       totalCount: [{ $count: "count" }],
     }
     ```
3. Execute via `this.fetcher.aggregate(SupportUserRepository.COLLECTION, pipeline)`
4. Map results to `ISupportUserDto[]` via a private `mapToDto` method that converts `_id` to string, maps subscription/business subdocuments

**CRITICAL: ObjectId-to-string join.** The `subscription.userId` field is a string representation of the user's `_id` ObjectId. The `$lookup` must use `let: { odUserId: { $toString: "$_id" } }` with `$expr: { $eq: ["$userId", "$$odUserId"] }` to handle the type mismatch. Similarly, `businesses` collection uses ObjectId `_id` and user entity stores `businessIds` as ObjectId array -- verify this during implementation.
