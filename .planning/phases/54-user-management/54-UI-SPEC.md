## Interaction Contracts

### Dashboard Metric Cards

1. Five cards render in a responsive grid: `grid gap-4 md:grid-cols-5` (stacks to single column on mobile)
2. Each card shows: icon (top-right, `h-4 w-4 text-muted-foreground`), label (`text-sm font-medium`), count (`text-2xl font-bold`), sublabel (`text-xs text-muted-foreground`)
3. Cards are clickable -- entire card is a `Link` to `/support/users` with the appropriate filter query param
4. Hover state: card border transitions to `border-primary` with `transition-colors`
5. Card icons: Total Users = `Users`, Active Trials = `Clock`, Active Subscriptions = `CreditCard`, Expired = `XCircle`, Canceled = `Ban`
6. **Loading state**: 5 `Skeleton` cards matching card dimensions
7. **Error state**: `Alert` with "Unable to load metrics. Please try again." and Retry button

### User List Page

#### Search

1. Search input with `Search` icon (left-positioned, `absolute left-3 top-1/2 h-4 w-4 -translate-y-1/2 text-muted-foreground`)
2. Placeholder: "Search by name or email..."
3. Debounce: 300ms before triggering API call
4. Search clears pagination back to page 1

#### Role Filter Tabs

1. `Tabs` component with three triggers: "All", "Support", "Customer"
2. Selecting a tab updates the role filter query param and resets to page 1
3. Default: "All" selected

#### Subscription Status Filter

1. `Select` component to the right of the role tabs
2. Options: "All statuses", "Trialing", "Active", "Past Due", "Canceled", "Expired"
3. Selecting a status updates the filter query param and resets to page 1
4. Default: "All statuses" selected

#### Table

1. Columns: Name (font-medium), Email, Subscription Status (badge), Created (formatted date)
2. Entire row is clickable -- navigates to `/support/users/:id`
3. Row hover state: `hover:bg-muted/50` (standard table row hover)
4. No action menu (MoreHorizontal removed -- this phase is read-only; Phase 55 adds actions)
5. **Loading state**: 10 `Skeleton` rows matching table row height
6. **Empty state**: Centered message inside table card: "No users found" heading + "No users match your current search or filters. Try adjusting your search term or clearing filters." body

#### Pagination

1. Below the table card, right-aligned
2. "Page X of Y" text center, "Previous" and "Next" buttons flanking
3. Buttons disabled at boundaries (page 1 = Previous disabled, last page = Next disabled)
4. Page size: 20 users per page (server-side)

### User Detail Page

#### Layout

1. Back button: `Button variant="ghost" size="icon"` with `ArrowLeft` icon, links to `/support/users`
2. Header: User name as `text-3xl font-bold tracking-tight`, email below as `text-muted-foreground`
3. Role badges rendered inline next to user name (if support user)
4. Four section cards in a 2-column grid on desktop (`grid gap-6 md:grid-cols-2`), single column on mobile

#### Profile Card

1. Title: "Profile"
2. Fields (label: value pairs, vertical stack): Display Name, Email, Account Created (from Firebase), Last Sign-In (from Firebase)
3. Labels: `text-sm text-muted-foreground`, values: `text-sm font-medium`

#### Business Card

1. Title: "Business"
2. If business exists: Business Name, Trade Type
3. If no business: "No business associated" in `text-sm text-muted-foreground`

#### Subscription Card

1. Title: "Subscription"
2. Status badge (using `SubscriptionBadge` component) at top-right of card header
3. Fields: Status, Trial Start, Trial End, Subscription Start, Current Period End, Canceled At (only if canceled)
4. Date formatting: `DateTime.fromISO(date).toLocaleString(DateTime.DATE_MED)` via Luxon
5. If no subscription: "No subscription" in `text-sm text-muted-foreground`

#### Roles Card

1. Title: "Roles"
2. List of role badges using `RoleBadge` component
3. If no support roles: "No support roles assigned" in `text-sm text-muted-foreground`
4. This card is read-only in Phase 54. Phase 55 adds grant/revoke buttons here.

#### Loading State

1. All four cards render as `Skeleton` blocks
2. Header name and email as `Skeleton` lines

#### Error State

1. If user not found (404): Navigate to `/support/users` with toast "User not found"
2. If network error: `Alert` with "Unable to load user details. Please try again." and Retry button

### Filter-to-URL Synchronization

1. All filter state (search, role tab, subscription status, page) is stored in URL search params
2. Format follows D-01 convention: `?search=john&filter:role.type:eq=support&filter:subscription.status:eq=active&page=2`
3. Navigating to `/support/users` from a metric card pre-populates the appropriate filter
4. Browser back/forward preserves filter state
