## Interaction States

### Welcome Section (Dashboard Top)

| State | Condition | Visual |
|-------|-----------|--------|
| Visible (new user) | Onboarding progress record shows incomplete checklist (not all items done) | Welcome greeting + checklist card rendered at top of dashboard |
| Hidden (returning user) | All checklist items complete | Welcome section not rendered at all -- dashboard shows standard content only |
| Loading | Onboarding progress data still fetching | Welcome section not rendered (avoid layout shift -- show once data arrives) |

### Checklist Items

| State | Condition | Visual |
|-------|-----------|--------|
| Incomplete | Onboarding progress record shows item not done | Empty `Circle` icon (text-muted-foreground/40), title in `text-foreground`, description in `text-muted-foreground`, `ChevronRight` arrow on right |
| Complete | Onboarding progress record shows item done | `Check` icon inside filled circle (`bg-primary/10 text-primary`), title with `line-through text-muted-foreground`, description hidden, no arrow |
| Hover (incomplete) | Mouse over incomplete row | `bg-muted/50` row background highlight, cursor-pointer |
| Click (incomplete) | User clicks incomplete row | Navigate to relevant page (`/jobs` or `/quotes`) |
| Click (complete) | User clicks complete row | No action (row is not interactive when complete) |

### Widget Disappearance

| State | Condition | Visual |
|-------|-----------|--------|
| Partial complete | One item done, one remaining | Widget stays visible, completed item shown with strikethrough |
| All complete | Both items marked done in onboarding progress | Widget disappears entirely on next render/refetch -- no animation, no celebration (per D-08) |

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| Mobile (< 768px) | Welcome section full-width with standard page padding (`px-4`). Card padding `px-4 py-4`. Checklist items stack naturally (single column). |
| Desktop (>= 768px) | Welcome section follows dashboard content max-width. Card padding `px-6 py-6`. Layout unchanged -- checklist is already single-column by design. |

## Accessibility Notes

- Welcome greeting uses `<h1>` (or appropriate heading level within dashboard page hierarchy)
- Checklist card heading uses `<h2>`
- Incomplete checklist items are wrapped in `<Link>` (or `<a>`) elements for keyboard navigation
- Complete checklist items are `<div>` elements (non-interactive)
- Completed checkmark has `aria-label="Complete"` for screen readers
- Incomplete item icon has `aria-label="Not started"` for screen readers
- Checklist container uses `<ul>` with `<li>` for each item, providing list semantics
- Strikethrough text on completed items supplemented with visually hidden "(completed)" text for screen readers
