    Replace the stub event handling in `handleStripeWebhook` method. After `constructEvent` succeeds, enqueue the event and return void (implicit 200):

    ```typescript
    @Post("stripe")
    @Public()
    async handleStripeWebhook(
      @Req() req: RawBodyRequest<Request>,
      @Headers("stripe-signature") signature: string,
    ): Promise<void> {
      const webhookSecret = this.configService.get<string>("STRIPE_WEBHOOK_SECRET");
      let event: Stripe.Event;

      try {
        event = this.stripe.webhooks.constructEvent(
          req.rawBody,
          signature,
          webhookSecret,
        );
      } catch (error) {
        this.logger.error(`Webhook signature verification failed: ${error.message}`);
        throw new BadRequestException("Webhook signature verification failed");
      }

      try {
        await this.queueProducer.enqueueStripeWebhook(event);
        this.logger.log(`Enqueued Stripe event: ${event.type}`, { eventId: event.id });
      } catch (error) {
        // Still return 200 to Stripe even if enqueue fails (per Pitfall 7 from RESEARCH.md)
        // Log the error for investigation but do NOT throw
        this.logger.error(`Failed to enqueue Stripe event: ${event.type}`, error.stack, { eventId: event.id });
      }
    }
    ```

    Key points:
    - Signature verification failure throws `BadRequestException` (HTTP 400) -- this is correct behavior, Stripe should retry
    - Enqueue failure is caught silently with a log -- returns 200 to prevent Stripe retries (per Pitfall 7)
    - Import `BadRequestException` from `@nestjs/common`
    - Import `QueueProducer` from `@queue/services/queue-producer.service`
    - Remove the old `{ received: true }` return value -- void return gives implicit 200
