## Interaction States

### PaywallGuard

| State | Condition | Visual |
|-------|-----------|--------|
| Loading | Subscription data not yet fetched (`isLoading === true`) | Centered `Loader2` spinner on `bg-background` (same loading screen as OnboardingGuard -- seamless transition) |
| Support bypass | `user.supportRoles.length > 0` | Renders `<Outlet />` regardless of subscription status |
| Active | Subscription status is `trialing` or `active` | Renders `<Outlet />` (pass-through to DashboardLayout) |
| Trial expired | Subscription status `canceled` AND `canceledAt` close to `trialEnd` | Renders `PaywallPage` with trial-expired variant |
| Payment failed | Subscription status `past_due` | Renders `PaywallPage` with payment-failed variant |
| Canceled | Subscription status `canceled` AND `canceledAt` not near `trialEnd` | Renders `PaywallPage` with canceled variant |
| No subscription | `getSubscription` returns null (404) | Renders `PaywallPage` with trial-expired variant (fallback) |

### PaywallPage -- Trial Expired Variant

| State | Trigger | Visual |
|-------|---------|--------|
| Default | Guard renders PaywallPage with `variant="trial-expired"` | Card with headline, body copy, Subscribe + Manage Billing buttons, log out link |
| Subscribe loading | User clicks "Subscribe" | Button shows `Loader2` spinner, text changes to "Redirecting...", disabled state |
| Subscribe redirect | Checkout URL received from POST /v1/subscription/checkout | `window.location.href` redirect to Stripe (page unloads) |
| Subscribe error | Checkout API fails | Sonner error toast: "Unable to start checkout. Please try again." Button re-enables. |
| Manage Billing click | User clicks "Manage Billing" | Opens Stripe Billing Portal URL in new tab |
| Manage Billing error | Portal API fails | Sonner error toast: "Unable to open billing. Please try again." |
| Log out click | User clicks "Log out" | Firebase sign-out, redirect to `/` (landing page) |

### PaywallPage -- Payment Failed Variant

| State | Trigger | Visual |
|-------|---------|--------|
| Default | Guard renders PaywallPage with `variant="payment-failed"` | Card with headline, body copy, Manage Billing button only (no Subscribe), log out link |
| Manage Billing click | User clicks "Manage Billing" | Opens Stripe Billing Portal URL in new tab |
| Manage Billing error | Portal API fails | Sonner error toast: "Unable to open billing. Please try again." |
| Log out click | User clicks "Log out" | Firebase sign-out, redirect to `/` |

### PaywallPage -- Canceled Variant

| State | Trigger | Visual |
|-------|---------|--------|
| Default | Guard renders PaywallPage with `variant="canceled"` | Card with headline, body copy, Subscribe + Manage Billing buttons, log out link |
| Subscribe loading | User clicks "Subscribe" | Button shows `Loader2` spinner, text changes to "Redirecting...", disabled state |
| Subscribe redirect | Checkout URL received | `window.location.href` redirect to Stripe |
| Subscribe error | Checkout API fails | Sonner error toast: "Unable to start checkout. Please try again." |
| Manage Billing click | User clicks "Manage Billing" | Opens Stripe Billing Portal URL in new tab |
| Log out click | User clicks "Log out" | Firebase sign-out, redirect to `/` |

## Layout Specifications

### PaywallPage (Full Viewport -- matches onboarding wizard layout)

```
PaywallPage (min-h-screen bg-gradient-to-br from-background to-muted/30)
  div (flex flex-col items-center justify-center min-h-screen px-4 py-12)
    Trade Flow Logo (text-lg font-semibold text-foreground mb-8)
      "Trade Flow" (text only, matching LandingHeader and OnboardingWizard treatment)

    Card (w-full max-w-[480px])
      CardHeader (px-6 pt-6 pb-0)
        h1 (text-lg font-semibold): variant headline
        p (text-sm text-muted-foreground mt-2): variant body copy
      CardContent (px-6 py-6)
        [empty -- body copy is in CardHeader subtitle]
      CardFooter (px-6 pb-6 pt-0 flex flex-col gap-2)
        [Trial expired / Canceled variants:]
          Button (w-full, variant="default")
            CreditCard icon (h-4 w-4 mr-2)
            "Subscribe -- GBP 6/month"
          Button (w-full, variant="outline")
            ExternalLink icon (h-4 w-4 mr-2)
            "Manage Billing"

        [Payment failed variant:]
          Button (w-full, variant="default")
            ExternalLink icon (h-4 w-4 mr-2)
            "Manage Billing"

    button (text-sm text-muted-foreground hover:text-foreground mt-6 underline-offset-4 hover:underline)
      "Log out"
```

Note: CardContent section is structurally present but visually empty since body copy sits in the CardHeader subtitle. This keeps the card compact and the message prominent. If the card feels too sparse, CardContent can be omitted entirely and CardFooter padding adjusted.

### PaywallGuard Loading Screen

```
div (flex items-center justify-center min-h-screen bg-background)
  Loader2 (h-8 w-8 animate-spin text-muted-foreground)
```

Same loading screen pattern as OnboardingGuard and Phase 36 suspense fallback -- seamless transition from app loading to either paywall or application.

## Accessibility Notes

- Paywall card heading uses `<h1>` (single h1 per page)
- Subscribe and Manage Billing buttons have descriptive text labels (no icon-only buttons)
- Loading state spinner has `aria-label="Loading"` or visually hidden text
- Log out is a `<button>` element (not a link) since it performs an action
- Manage Billing button has `aria-label="Manage Billing (opens in new tab)"` to communicate new-tab behavior
- Error toasts via Sonner are announced by screen readers (existing Sonner accessibility)
- Card uses semantic structure: heading + description + action buttons
- Keyboard: Tab cycles through Subscribe, Manage Billing, Log out. Enter/Space activates.
