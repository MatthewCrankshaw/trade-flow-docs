In `permission.guard.ts`:

2. Remove import of `ForbiddenError` from `@core/errors/forbidden-error.error`.
3. Remove import of `ErrorCodes` from `@core/errors/error-codes.enum`.
4. Replace both `throw new ForbiddenError(ErrorCodes.ACTION_FORBIDDEN, ...)` calls with:
```typescript
throw new ForbiddenException({
  errors: [
    {
      code: "ACTION_FORBIDDEN",
      message: "Access denied",
      details: `Missing required permission: ${requiredPermissions.join(", ")}`,
    },
  ],
});
```

Use the string `"ACTION_FORBIDDEN"` directly (not the enum) since we are removing the ErrorCodes import -- this keeps the response body shape consistent with `createHttpError()` output while using NestJS's built-in exception that the framework catches and maps to 403 automatically.

The response body structure `{ errors: [{ code, message, details }] }` matches the `IResponse` error shape used throughout the app (same as what `createHttpError` produces for ForbiddenError), so frontend error parsing in `RevokeRoleDialog` (`error.data.errors[0].details`) continues to work.
