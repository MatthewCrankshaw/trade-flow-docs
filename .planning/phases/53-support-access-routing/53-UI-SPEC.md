## Interaction Contracts

### Login Flow (Modified)

1. User enters credentials on LoginPage
2. Firebase auth succeeds
3. Fetch user data via `useGetCurrentUserQuery()`
4. Check `location.state?.from` -- if set, navigate there (D-03)
5. Check `user.supportRoles.length > 0` -- if true, navigate to `/support` (D-01, D-02)
6. Otherwise navigate to `/dashboard`

**Loading state**: Show existing login loading indicator while user data fetches.
**Error state**: If user data fetch fails, show toast "Unable to determine your account type. Please try again." and remain on login page.

### SupportGuard Behaviour

1. Read user from `useGetCurrentUserQuery()`
2. While loading: render a centered spinner or skeleton (follow existing guard patterns)
3. If `supportRoles.length > 0`: render `<Outlet />`
4. If not: render `<Navigate to="/dashboard" replace />`

**No error banner, no flash of content** -- guard renders nothing until user data is resolved.

### Sidebar Navigation (Modified)

Support users see:
```
Support
  Dashboard       /support
  Users           /support/users
  Businesses      /support/businesses
```

Business users see (unchanged):
```
Main
  Dashboard       /dashboard
  Jobs            /jobs
  Customers       /customers
  Quotes          /quotes
  Estimates       /estimates
  Items           /items

Business
  My Business     /business
  Settings        /settings
```

Icons for support nav items:
- Dashboard: `LayoutDashboard` (same as business dashboard)
- Users: `Users`
- Businesses: `Building2`
