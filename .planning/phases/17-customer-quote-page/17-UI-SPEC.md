## Page Layout Contract

### Public Quote Page (`/quote/view/:token`)

**Focal point:** The quote card itself is the single focal point -- centered on a neutral background with no competing navigation or chrome.

**Route placement:** Outside `AuthenticatedLayout` in `App.tsx`. No sidebar, no header, no Trade Flow app chrome.

**Page structure:**

```
┌─────────────────────────────────────────────┐
│  bg-background full viewport                │
│                                             │
│  ┌─────────────────────────────────────┐    │
│  │  Quote Card (max-w-3xl mx-auto)     │    │
│  │                                     │    │
│  │  ┌─ Status Banner (if responded) ─┐ │    │
│  │  │ "You accepted this quote on.." │ │    │
│  │  └────────────────────────────────┘ │    │
│  │                                     │    │
│  │  ┌─ Business Header ─────────────┐ │    │
│  │  │ Business Name                 │ │    │
│  │  │ Business address/contact      │ │    │
│  │  └────────────────────────────────┘ │    │
│  │                                     │    │
│  │  ┌─ Quote Details ───────────────┐ │    │
│  │  │ Quote #  │  Date  │  Valid To │ │    │
│  │  └────────────────────────────────┘ │    │
│  │                                     │    │
│  │  ┌─ Customer Details ────────────┐ │    │
│  │  │ Customer name, address        │ │    │
│  │  └────────────────────────────────┘ │    │
│  │                                     │    │
│  │  ┌─ Line Items ──────────────────┐ │    │
│  │  │ Desktop: Table layout         │ │    │
│  │  │ Mobile: Stacked cards         │ │    │
│  │  └────────────────────────────────┘ │    │
│  │                                     │    │
│  │  ┌─ Totals ──────────────────────┐ │    │
│  │  │ Subtotal         $X,XXX.XX    │ │    │
│  │  │ Tax (GST)          $XXX.XX    │ │    │
│  │  │ ────────────────────────────  │ │    │
│  │  │ Total            $X,XXX.XX    │ │    │
│  │  └────────────────────────────────┘ │    │
│  │                                     │    │
│  │  ┌─ Validity ────────────────────┐ │    │
```

**Card sizing:** `max-w-3xl` centered with `mx-auto`, padded with `p-4 sm:p-6` on the outer wrapper. Outer wrapper uses `min-h-screen bg-background py-6 sm:py-12`.

**Responsive breakpoint:** 768px (`md:`) — table on desktop, stacked cards on mobile. Follows existing DataView pattern.

## State Contracts

### Loading State

Skeleton matching card layout sections:

| Section | Skeleton |
|---------|----------|
| Status banner | Not shown during loading |
| Business header | `Skeleton` h-6 w-48 (name) + `Skeleton` h-4 w-64 (address) |
| Quote details | 3x `Skeleton` h-4 w-24 in a row |
| Customer details | `Skeleton` h-4 w-40 (name) + `Skeleton` h-4 w-56 (address) |
| Line items | Desktop: 4 table rows of `Skeleton` h-4. Mobile: 3 card skeletons |
| Totals | 3x `Skeleton` h-4 w-32 right-aligned |

### Error States

| Condition | Display | Copy |
|-----------|---------|------|
| Invalid/unknown token | Centered error card | Heading: "Quote not found". Body: "This quote link is not valid. It may have been removed or the link is incorrect." |
| Expired token | Centered error card with business name | Heading: "Quote expired". Body: "This quote from {businessName} has expired. Please contact them for an updated quote." |
| Revoked token | Centered error card with business name | Heading: "Quote no longer available". Body: "This quote from {businessName} is no longer available. Please contact them for more information." |
| Network error | Centered error card | Heading: "Something went wrong". Body: "We couldn't load this quote. Please check your internet connection and try again." Retry button: "Reload Quote" |

Error card layout: centered on page (`max-w-md mx-auto`), icon (`AlertCircle` or `FileX` at `h-12 w-12 text-muted-foreground`), heading, body text, optional action button. Follows existing empty state pattern spacing (`py-12`).

### Already-Responded States

| Status | Banner style | Copy |
|--------|-------------|------|
| Accepted | `bg-success/10 text-success-foreground` border-l-4 `border-success` | "You accepted this quote on {date}" |
| Rejected | `bg-warning/10 text-warning-foreground` border-l-4 `border-warning` | "You declined this quote on {date}" |

Banner positioned at top of card content, before business header. Full quote content remains visible below.

## Line Items Display Contract

### Desktop (md: and above) — Table Layout

| Column | Alignment | Content |
|--------|-----------|---------|
| Item | left | Item name + type badge (`Badge variant="outline"` with text-xs: "Material", "Labour", "Fee") |
| Qty | right | Quantity number |
| Unit Price | right | Formatted currency via `useCurrency` |
| Total | right | Formatted currency via `useCurrency`, `font-semibold` |

Bundles display as single rolled-up lines. Customer does not see bundle components.

### Mobile (below md:) — Stacked Card Layout

Follow `QuoteLineItemsCardList` pattern. Each line item as a card:

```
┌──────────────────────────────┐
│ Item Name          [Labour]  │
│ 2 × $85.00           $170.00│
└──────────────────────────────┘
```

- Item name: `font-semibold`
- Type badge: `Badge variant="outline"` size small
- Quantity x unit price: `text-sm text-muted-foreground`
- Line total: `text-sm font-semibold`, right-aligned

## View Tracking Contract

### API side (firstViewedAt update)

- On `GET /v1/public/quote/:token`, if `firstViewedAt` is null on the token record, set it to current timestamp
- No additional API endpoint needed — piggybacks on existing public quote fetch

### Tradesperson-facing viewed indicator

- Location: `QuoteActionStrip.tsx` on the quote detail page (tradesperson's authenticated view)
- Component: `Badge variant="outline"` with `Eye` icon (`h-3 w-3`)
- Text: "Viewed" + formatted timestamp (e.g., "15 Mar at 2:34 pm")
- Style: `text-xs text-muted-foreground` within badge
- Condition: Only shown when `firstViewedAt` is not null on the quote token
- Position: Inline with existing action buttons in the action strip
