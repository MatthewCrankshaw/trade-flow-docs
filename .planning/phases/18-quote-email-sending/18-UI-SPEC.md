## SendQuoteDialog Layout Contract

### Dialog sizing

`sm:max-w-lg` (medium dialog -- standard form with 3 fields including a rich text area).

### Structure

```
+------------------------------------------+
|  Send Quote                         [x]  |
|  Review and send this quote to your      |
|  customer.                               |
|                                          |
|  To                                      |
|  [john@example.com                    ]  |
|  ! No email on file for this customer    |
|  [ ] Save email to customer record       |
|                                          |
|  Subject                                 |
|  [Quote Q-0042 for Kitchen Renovation ]  |
|                                          |
|  Message                                 |
|  +--------------------------------------+|
|  | B  I  Link  |                        ||
|  |--------------------------------------|
|  | Hi John,                             ||
|  |                                      ||
|  | Please find your quote Q-0042 for    ||
|  | Kitchen Renovation.                  ||
|  |                                      ||
|  | Kind regards,                        ||
|  | Mike                                 ||
|  +--------------------------------------+|
|                                          |
|              [Cancel]  [Send Quote]      |
+------------------------------------------+
```

### Field specifications

| Field | Type | Pre-fill source | Validation |
|-------|------|----------------|------------|
| To | Email input (`Input`) | `customer.email` or empty if null | Required, valid email format |
| Subject | Text input (`Input`) | Default template subject with variables resolved | Required, non-empty |
| Message | Rich text (`RichTextEditor`) | Default template body with variables resolved | Required, non-empty |

### Auto-focus

Per D-05: focus lands on the message body (Tiptap editor) when the dialog opens. Use `autofocus: "end"` on the Tiptap editor instance.

### Warning state (no customer email)

When `customer.email` is null:
- To field renders empty
- Below To field: `Alert` component with `warning` variant -- icon `AlertTriangle` (`h-4 w-4`) + text "No email on file for this customer"
- Below alert: Checkbox row -- `[ ] Save email to customer record` (`text-sm font-medium`)
- Checkbox always visible when To field has a value (regardless of whether customer had an email), unchecked by default, checked by default when customer had no email

### Send button states

| State | Button |
|-------|--------|
| Ready | `<Button>Send Quote</Button>` (default variant) with `Send` icon (lucide) `h-4 w-4` |
| Submitting | `<Button disabled>Sending...</Button>` with loading spinner |
| To field empty | `<Button disabled>Send Quote</Button>` |

### Post-send behavior (D-06)

1. `toast.success("Quote sent to john@example.com")` -- includes the actual recipient email
2. Dialog closes via `onOpenChange(false)`
3. RTK Query cache invalidation refreshes quote detail -- status badge updates to "Sent", sentAt timestamp appears

## Rich Text Editor Contract

### Component: `RichTextEditor`

Location: `src/components/ui/rich-text-editor.tsx`

### Toolbar

Minimal toolbar with 3 formatting options only:

| Button | Icon | Toggle | Keyboard shortcut |
|--------|------|--------|-------------------|
| Bold | `Bold` (lucide) `h-4 w-4` | `toggleBold` | Cmd+B |
| Italic | `Italic` (lucide) `h-4 w-4` | `toggleItalic` | Cmd+I |
| Link | `Link` (lucide) `h-4 w-4` | `toggleLink` (prompts for URL) | Cmd+K |

### Toolbar styling

- Container: `flex gap-1 border-b border-border px-2 py-1`
- Toggle buttons: `Button variant="ghost" size="icon"` with `h-8 w-8` sizing
- Active state: `bg-accent text-accent-foreground` (standard shadcn toggle pattern)
- Inactive state: `text-muted-foreground`

### Editor area styling

- Container: `rounded-md border border-input bg-background`
- Content area: `prose prose-sm max-w-none p-3 min-h-[120px] focus-within:ring-2 focus-within:ring-ring focus-within:ring-offset-2`
- Placeholder: `text-muted-foreground` -- "Write your message..."

### Tiptap configuration

- Extensions: `StarterKit` (with `heading: false`, `codeBlock: false`, `blockquote: false`), `Link` (with `openOnClick: false`), `Placeholder`
- Output: HTML string via `editor.getHTML()`

## State Contracts

### Loading State (SendQuoteDialog)

Dialog opens immediately with fields pre-filled from local data (customer email, resolved template). No loading skeleton needed -- all data is available from the already-loaded quote detail page context.

If business settings (template) have not been fetched yet, use the system default template (D-09) as fallback.

### Error States

| Condition | Display | Copy |
|-----------|---------|------|
| SendGrid delivery failure | Toast error | `toast.error("Failed to send quote. Please try again.")` |
| Network error on send | Toast error | `toast.error("Could not connect to the server. Check your connection and try again.")` |
| Invalid email (server-side) | Toast error | `toast.error("The email address is not valid. Please check and try again.")` |
| Business settings save failure | Toast error | `toast.error("Failed to save template. Please try again.")` |

All errors are toast-based. The dialog remains open on send failure so the user can retry without re-entering data.

### Empty States

| Location | Condition | Display |
|----------|-----------|---------|
| Business settings template fields | Fields are empty (no template configured) | Placeholder text in input/textarea showing the system default values in `text-muted-foreground` |

No full empty state illustration needed -- the settings form is always visible with placeholder defaults.

## Customer-Facing Email Design Contract

Per D-10 and D-11, the HTML email is rendered server-side with Maizzle + Tailwind CSS. This contract defines the visual structure for the email template.

### Layout

Centered card on grey background, max-width 600px (email standard).

### Structure

```
+------------------------------------------+
|  {businessName}                           |
|  ----------------------------------------|
|                                          |
|  {tradesperson's personal message}        |
|  (sanitized HTML from Tiptap)            |
|                                          |
|          [View Quote Online]              |
|                                          |
|  Quote: {quoteNumber}                    |
|  Date: {quoteDate}                       |
|  Total: {totalIncTax}                    |
|                                          |
|        Sent via Trade Flow               |
+------------------------------------------+
```

### Email typography (inline styles, not Tailwind tokens)

| Element | Style |
|---------|-------|
| Business name | 20px, bold, dark grey (#1a1a1a) |
| Message body | 16px, normal weight, dark grey (#374151), line-height 1.6 |
| CTA button | 16px, semibold, white (#ffffff) on blue (#2563eb), padding 12px 32px, border-radius 6px |
| Quote summary labels | 14px, normal weight, grey (#6b7280) |
| Quote summary total | 14px, semibold, dark grey (#1a1a1a) |
| Footer | 12px, light grey (#9ca3af), centered |

### CTA button

| Property | Value |
|----------|-------|
| Label | "View Quote Online" |
| Background | #2563eb (blue-600 equivalent) |
| Text | #ffffff, 16px, semibold |
| Padding | 12px 32px |
| Border radius | 6px |
| Link | `{appUrl}/quote/view/{token}` |
| VML fallback | MSO conditional for Outlook rendering |

### Sanitization allowlist (server-side)

Only these HTML tags are allowed in the message body injection: `p`, `br`, `strong`, `em`, `a`, `ul`, `ol`, `li`. All other tags are stripped.
