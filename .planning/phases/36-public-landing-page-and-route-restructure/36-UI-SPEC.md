| Token | Value | Tailwind | Usage in Phase 36 |

## Typography

All values from the established Trade Flow type system (consistent with Phase 32):

| Role | Size | Weight | Line Height | Tailwind | Phase 36 Usage |
|------|------|--------|-------------|----------|----------------|
| Body | 14px | 400 (regular) | normal | `text-sm` | Feature card descriptions, pricing card features, footer text |
| Label | 12px | 400 (regular) | normal | `text-xs` | Price period label, fine print ("no card required") |
| Heading | 18px | 600 (semibold) | none | `text-lg font-semibold` | Feature card titles, pricing card title, price display ("GBP 6") |
| Display | 36px | 600 (semibold) | 1.2 | `text-4xl font-semibold tracking-tight` | Hero headline |

Additional patterns used:
- Hero subheadline: `text-lg text-muted-foreground` (18px, 400) -- secondary copy below headline, uses Body weight at Heading size
- Price display: `text-lg font-semibold` (18px, 600) -- the plan price "GBP 6", uses Heading role
- Muted descriptions: `text-sm text-muted-foreground` -- secondary copy throughout landing page
- Fine print: `text-xs text-muted-foreground` -- "No card required", "Cancel any time"

## Color

All values from the established Trade Flow colour system (consistent with Phase 32):

| Role | Token | OKLCH | Usage in Phase 36 |
|------|-------|-------|-------------------|
| Dominant (60%) | `background` | `oklch(0.99 0 0)` | Landing page canvas, hero background, sections background |
| Secondary (30%) | `card` | `oklch(1 0 0)` | Feature highlight cards, pricing card surface |
| Accent (10%) | `primary` | `oklch(0.31 0.09 245)` | See reserved list below |
| Destructive | `destructive` | `oklch(0.64 0.21 25)` | Not used in Phase 36 (no destructive actions) |

Accent (`primary`) reserved for:
- "Start Free Trial" hero CTA button (`bg-primary text-primary-foreground`)
- "Start Free Trial" pricing card CTA button (`bg-primary text-primary-foreground`)
- Feature card icons (`text-primary`) -- accent colour draws attention to capability icons

Secondary accent usage:
- "Log in" header button uses `variant="outline"` (secondary treatment, not primary accent)
- `muted` background for alternate landing page sections if visual separation needed
- `accent` for hover states on interactive navigation elements

## Interaction States

### Landing Page Auth Check

| State | Condition | Visual |
|-------|-----------|--------|
| Loading | Firebase `onAuthStateChanged` not yet resolved | Centered `Loader2` spinner (h-8 w-8 animate-spin text-muted-foreground) on white background |
| Unauthenticated | Auth state resolved, no user | Full landing page renders |
| Authenticated | Auth state resolved, user present | `navigate('/dashboard')` redirect, user never sees landing page |

### Hero CTA

| State | Trigger | Visual |
|-------|---------|--------|
| Default | Page loaded, unauthenticated | `Button` primary: "Start Free Trial" with `ArrowRight` icon suffix |
| Hover | Mouse over | Standard primary button hover state (existing shadcn treatment) |
| Click | User clicks | Navigate to `/login?mode=signup` |

### Landing Header Navigation

| State | Trigger | Visual |
|-------|---------|--------|
| Default | Page loaded | "Log in" outline button + "Start Free Trial" primary button |
| Mobile | Viewport < 768px | Buttons stack or reduce to icon-only as needed |

### Suspense Fallback

| State | Condition | Visual |
|-------|-----------|--------|
| Loading | LandingPage chunk downloading | Centered spinner: `Loader2` (h-8 w-8 animate-spin text-muted-foreground) on `bg-background` full viewport |

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| Mobile (< 768px) | Hero headline `text-4xl`. Feature cards stack vertically (single column). Pricing card full-width. Header buttons reduce to `size="sm"`. |

## Bundle Isolation Contract

Per decisions D-09, D-10, D-11:

| Rule | Enforcement |
|------|-------------|

## Accessibility Notes

- Landing page uses semantic HTML: `<header>`, `<main>`, `<section>`, `<footer>`
- All CTA buttons have descriptive text (no icon-only buttons without aria-label)
- Feature cards use `<h3>` headings for card titles within the features `<section>`
- Hero headline is `<h1>` -- only one h1 per page
- Pricing section heading uses `<h2>`
- "Log in" and "Start Free Trial" nav buttons are accessible via keyboard tab order
- Auth loading spinner has `aria-label="Loading"` or visually hidden text
- Skip-to-content link not required (landing page has minimal nav, content is immediately visible)
