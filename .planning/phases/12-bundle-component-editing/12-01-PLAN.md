<objective>
Extend the API update endpoint to accept and validate bundle components, and align the frontend type.

Purpose: Currently updating a bundle item can only change priceStrategy and bundlePrice. This plan enables the full component array to be sent in update requests, with the same validation rules as creation (active items, non-bundle, 1-100 count, quantity > 0). The frontend type is also updated so TypeScript allows sending components.

Output: API accepts components on PATCH /items/:id, validates them, persists via full replacement. Frontend UpdateItemRequest type includes components.
</objective>
