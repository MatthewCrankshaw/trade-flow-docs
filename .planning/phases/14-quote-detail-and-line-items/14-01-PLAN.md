    - `update(id: string, updates: { quantity?: number; unitPrice?: number; lineTotal?: number; discountAmount?: number }): Promise<IQuoteLineItemDto>` -- use `findOneAndUpdate` with `$set` on provided fields, converting price fields to major units.     **4. Add updateLineItem and deleteLineItem to QuoteUpdater** (`trade-flow-api/src/quote/services/quote-updater.service.ts`):

    `updateLineItem(authUser, quoteId, lineItemId, quantity?, unitPrice?, businessId)`:
    - Fetch quote via quoteRepository.findByIdOrFail, check access with accessControllerFactory
    - Fetch line item via quoteLineItemRepository.findByIdOrFail
    - If quantity changed: recalculate lineTotal = unitPrice * quantity (using Money arithmetic). If unitPrice also changed: use new unitPrice. Calculate discountAmount = originalUnitPrice - unitPrice (if unitPrice provided). Update via repository.update().
    - Re-fetch all line items via quoteLineItemRetriever, set on quote DTO, run quoteTotalsCalculator.calculateTotals, save totals to quote via quoteRepository (need a saveTotals method or reuse existing pattern).
    - Actually, simpler approach: after updating the line item in the repository, re-fetch the full quote (quote + line items) and recalculate totals. Follow the same pattern as addLineItem -- fetch quote, modify, recalculate, return.
    - Specifically: fetch quote, update the line item in DB, re-fetch line items into quote.lineItems, call quoteTotalsCalculator.calculateTotals, save updated totals back to quote via quoteRepository, return updated quote DTO.

    `deleteLineItem(authUser, quoteId, lineItemId, businessId)`:
    - Fetch quote via quoteRepository.findByIdOrFail, check access
    - Fetch line item via quoteLineItemRepository.findByIdOrFail
    - If line item has `parentLineItemId` set, reject with error -- cannot delete component line items individually (per CONTEXT: components are locked)
    - If line item is a bundle parent (type === 'bundle'), also delete all children via quoteLineItemRepository.deleteByParentLineItemId(lineItemId)
    - Delete the line item itself via quoteLineItemRepository.delete(lineItemId)
    - Re-fetch line items, recalculate totals, save, return updated quote

    NOTE: The QuoteRepository likely needs a method to save updated totals. Check if it has an `update` method. If not, add `updateTotals(quoteId: string, totals: IQuoteTotalsDto): Promise<void>` that uses `updateOne` with `$set` on the totals fields. The `addLineItem` flow currently appends to an in-memory collection and calculates -- but for update/delete we need to persist the recalculated totals. Check the existing `quoteRepository` methods and follow the established pattern. If `addLineItem` already persists somehow (via the totals calculator or elsewhere), follow the same approach.

    IMPORTANT: Look at how `addLineItem` persists -- it returns a DTO but may not save totals explicitly. The `QuoteTotalsCalculator.calculateTotals` just computes, it does not save. Check if there's a save step after addLineItem in the controller or somewhere. If addLineItem's "return quote" is just in-memory and the quote totals are computed on-the-fly at read time, then update/delete may just need to modify line items and the totals will auto-compute on next read. Investigate this carefully before adding unnecessary save logic.
