    - Class name: `CreateVisitTypesIndexesMigration`
    - Implements `IMigration` interface (from `@migration/interfaces/migration.interface.ts`)
    - `migrationName = "20260221000000-create-visit-types-indexes"`

    **`up(db: Db)` method -- create 2 indexes:**
    1. **Unique compound index** on `{ businessId: 1, name: 1 }` with `{ unique: true }`:
       - Prevents duplicate visit type names per business at the database level
       - Spans ALL statuses (including inactive) -- per decision, names are permanently reserved even after archiving. This means if "Quote" is archived, a new "Quote" cannot be created. The user would need to reactivate the existing one.
       - `await db.collection("visitTypes").createIndex({ businessId: 1, name: 1 }, { unique: true })`

    2. **Performance index** on `{ businessId: 1, status: 1 }`:
       - Supports the list endpoint which filters by businessId + status
       - `await db.collection("visitTypes").createIndex({ businessId: 1, status: 1 })`

    **`down(db: Db)` method -- drop both indexes:**
    - `await db.collection("visitTypes").dropIndex("businessId_1_name_1")`
    - `await db.collection("visitTypes").dropIndex("businessId_1_status_1")`

    **`isBootstrap()` method:**
    - Return `false` (this is not a bootstrap migration)

    Use the timestamp format consistent with existing migrations in the directory. Import `Db` from `mongodb` and `IMigration` from the migration interfaces.
