**item.schema.ts changes:**
Replace the `defaultTaxRate` field in `itemFormSchema` with `taxRateId`:
```
taxRateId: v.optional(v.string())
```
This is a simple optional string -- the tax rate ID selected from the dropdown. Remove the numeric validation logic that was on defaultTaxRate. The `ItemFormValues` type will automatically update via `v.InferOutput`.

**useItemForm.ts changes:**

1. Add `businessId` parameter usage: import `useGetTaxRatesQuery` from `@/services` and call it with `businessId` to fetch tax rates.

2. Update `defaultValues`:
   - Remove `defaultTaxRate: item?.defaultTaxRate?.toString() ?? ""`
   - Add `taxRateId`: For edit mode (`item` exists), use `item.taxRateId ?? ""`. For create mode, find the default tax rate from the fetched tax rates list (`taxRates?.find(tr => tr.isDefault)?.id ?? ""`) to pre-select it.

3. Update `handleSubmit` mapping:
   - Remove the `defaultTaxRate` mapping (the `Number.parseFloat` conversion)
   - Add: `taxRateId: values.taxRateId || null` (empty string becomes null for the API)
   - Remove `bundleConfig: null` from the mapped data -- bundles don't go through this hook

4. Return the `taxRates` data and loading state alongside `form` and `handleSubmit` so the form components can render the dropdown:
   ```typescript
   const { data: taxRates = [], isLoading: isTaxRatesLoading } = useGetTaxRatesQuery(businessId);
   // ... existing form setup ...
   return { form, handleSubmit, taxRates, isTaxRatesLoading };
   ```

5. Filter tax rates to only active ones for the dropdown options, but keep the full list for resolving archived tax rate names if needed:
   ```typescript
   const activeTaxRates = taxRates.filter(tr => tr.status === "enabled");
   return { form, handleSubmit, activeTaxRates, isTaxRatesLoading };
   ```
