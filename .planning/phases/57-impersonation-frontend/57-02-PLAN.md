**1. Create `trade-flow-ui/src/features/support/hooks/useImpersonation.ts`**

Follow the `useCustomerActions.ts` hook pattern. Create a hook that encapsulates the full impersonation lifecycle:

**Imports:**
- `useCallback` from `"react"`
- `useNavigate` from `"react-router-dom"`
- `useAppDispatch`, `useAppSelector` from `"@/store/hooks"`
- `startImpersonation`, `endImpersonation`, `selectImpersonation` from `"@/store/slices/impersonationSlice"`
- `useStartImpersonationMutation`, `useEndImpersonationMutation` from `"../api/impersonationApi"`
- `apiSlice` from `"@/services"`
- `toast` from `"@/lib/toast"`

**Interface for the return type:**
```typescript
interface UseImpersonationReturn {
  isActive: boolean;
  targetUser: { id: string; name: string; email: string } | null;
  sessionId: string | null;
  startSession: (targetUserId: string, targetUserName: string, targetUserEmail: string, reason: string) => Promise<void>;
  endSession: () => Promise<void>;
  isStarting: boolean;
  isEnding: boolean;
}
```

**Hook body:**
- Read impersonation state via `useAppSelector(selectImpersonation)`
- Get `useStartImpersonationMutation` and `useEndImpersonationMutation` mutation hooks
- Get `dispatch` from `useAppDispatch()` and `navigate` from `useNavigate()`

**`startSession` callback** (wrapped in `useCallback`):
1. Call `startMutation({ targetUserId, reason }).unwrap()`
2. On success: dispatch `startImpersonation({ token: result.token, sessionId: result.sessionId, targetUser: { id: targetUserId, name: targetUserName, email: targetUserEmail } })`
3. Then dispatch `apiSlice.util.resetApiState()` to clear all cached data
4. Then navigate to `"/"` (customer dashboard root)
5. On error: show `toast.error("Unable to start impersonation. Please try again.")`; re-throw the error

**`endSession` callback** (wrapped in `useCallback`):
1. If no `sessionId` in state, return early
2. Call `endMutation({ sessionId }).unwrap()`
3. On success: dispatch `endImpersonation()`, then dispatch `apiSlice.util.resetApiState()`, then navigate to `"/support"`, then show `toast.success("Impersonation session ended")`
4. On error: show `toast.error("Unable to end impersonation session. Please try again.")`; do NOT clear impersonation state (user can retry per D-12)

**Return:** `{ isActive: impersonation.active, targetUser: impersonation.targetUser, sessionId: impersonation.sessionId, startSession, endSession, isStarting, isEnding }`

Create the `src/features/support/hooks/` directory if it does not exist.

**1. Create `trade-flow-ui/src/features/support/components/ImpersonationBanner.tsx`**

Follow UI-SPEC and D-04/D-05/D-06 exactly. Create the component:

**Imports:**
- `AlertTriangle` from `"lucide-react"`
- `Loader2` from `"lucide-react"`
- `useImpersonation` from `"../hooks/useImpersonation"`

**Component:**
```typescript
export function ImpersonationBanner() {
  const { isActive, targetUser, endSession, isEnding } = useImpersonation();

  if (!isActive || !targetUser) {
    return null;
  }

  return (
    <div
      role="alert"
      aria-live="assertive"
      className="fixed top-0 left-0 right-0 z-[60] flex h-10 items-center justify-between bg-amber-500 px-4"
    >
      <div className="flex min-w-0 items-center gap-2">
        <AlertTriangle className="h-4 w-4 shrink-0 text-amber-950" />
        <span className="truncate text-sm font-medium text-amber-950">
          Impersonating: {targetUser.name}
        </span>
      </div>
      <button
        type="button"
        onClick={endSession}
        disabled={isEnding}
        aria-label="End impersonation and return to support dashboard"
        className="shrink-0 rounded-md bg-amber-950 px-3 text-sm font-medium text-amber-50 hover:bg-amber-900 disabled:opacity-50 h-7"
      >
        {isEnding ? (
          <span className="flex items-center gap-1">
            <Loader2 className="h-3 w-3 animate-spin" />
            Ending...
          </span>
        ) : (
          "Return to Support"
        )}
      </button>
    </div>
  );
}
```

Key details:
- `z-[60]` ensures it sits above Radix overlays (z-50) per UI-SPEC
- `role="alert"` and `aria-live="assertive"` for screen reader announcement
- `truncate` on the name text to handle long names
- `min-w-0` on the left container for proper flex truncation
- `shrink-0` on icon and button to prevent compression
- No close/dismiss button -- only "Return to Support" exits
- Loading state shows Loader2 spinner with "Ending..." text
- `disabled:opacity-50` during loading

Create the `src/features/support/components/` directory if it does not exist.
