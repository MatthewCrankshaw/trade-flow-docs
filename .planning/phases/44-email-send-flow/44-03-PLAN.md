Method `send(authUser: IUserDto, estimateId: string, request: SendEstimateRequest): Promise<IEstimateDto>`:

**Step 0 - Load and authorize:** Retrieve estimate via `estimateRetriever.findByIdOrFail(estimateId)`. Policy check via `this.estimatePolicy.canUpdate(authUser, estimate)` -- throws `ForbiddenError` if the user does not own the estimate's business.

**Step 1 - Determine send path** per D-TKN-08:
```
if estimate.status === DRAFT && estimate.revisionNumber === 1 -> sendType = INITIAL (first-ever send)
if estimate.status === DRAFT && estimate.revisionNumber > 1 -> sendType = INITIAL (first send of revised version), shouldCancelFollowups = true
if estimate.status IN [SENT, VIEWED, RESPONDED, SITE_VISIT_REQUESTED] -> sendType = RESEND
else -> throw InvalidRequestError(ErrorCodes.ESTIMATE_NOT_SENDABLE, "Estimate cannot be sent in its current status")
```

**Step 2 - Resolve or create token** per D-TKN-04:
```
rootId = estimate.rootEstimateId
existingToken = await documentTokenRepository.findActiveByDocumentId(rootId, "estimate")
if existingToken is null:
  token = await documentTokenCreator.create(rootId, request.to, "estimate")
else:
  token = existingToken
viewUrl = `${configService.get("APP_URL")}/estimate/${token.token}`
```

**Step 3 - Load settings and render:**
Import formatRange at file top: `import { formatRange } from "@estimate/utilities/format-range.utility";`
```
settings = await estimateSettingsRetriever.findByBusinessId(estimate.businessId)
estimateDate = DateTime.fromJSDate(estimate.createdAt).toFormat("d MMMM yyyy")
priceDisplay = formatRange(estimate.priceRange.low.total, estimate.priceRange.high.total, estimate.displayMode, "GBP")

  businessName: estimate.businessName (or resolve from business),

  viewUrl,
  estimateNumber: estimate.number,
  estimateDate,

  contingencyNote,
  uncertaintyNotes: estimate.uncertaintyNotes || undefined,
})
```

**Step 4 - Send via Resend:**
```
await emailSenderService.sendEmail({ to: request.to, subject: request.subject, html, from: resolvedFromEmail, senderName: businessName })
```

**Steps 5-8 (after successful send)** per D-AUDIT-08:
5. Create audit row: `estimateEmailSendCreator.create({ estimateId: estimate.id, rootEstimateId: estimate.rootEstimateId, revisionNumber: estimate.revisionNumber, businessId: estimate.businessId, type: sendType, to: request.to, subject: request.subject, html, sentAt: DateTime.now() })`
6. Transition: `estimateTransitionService.transition(estimate, EstimateStatus.SENT)`
7. Update estimate: `estimateRepository.updateSendFields(estimateId, { lastSentAt: DateTime.now(), lastSentTo: request.to })`

9. If `shouldCancelFollowups`: find previous revision via `estimateRepository.findOne({ rootEstimateId: estimate.rootEstimateId, revisionNumber: estimate.revisionNumber - 1 })` and call `followupCanceller.cancelAllFollowups(previousRevision.id, previousRevision.revisionNumber)`

**Step 10 - Save email (optional):** If `request.saveEmail === true`, update customer email via `customerUpdater`.

**Error handling** per D-SND-05: Wrap steps 5-9 in try/catch. If any fail after Resend succeeded (step 4), log at `logger.error` with full context and re-throw as 500. The email is already delivered.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-44-09 | Spoofing | POST /v1/estimates/:id/send | mitigate | `JwtAuthGuard` requires valid Firebase JWT; policy checks business ownership |
| T-44-10 | Tampering | SendEstimateRequest.message | mitigate | `sanitizeHtml()` allowlist strips all tags except `p, br, strong, em, a, ul, ol, li` before rendering into template |
| T-44-11 | Tampering | SendEstimateRequest.subject | mitigate | `@Matches(/estimate/i)` class-validator decorator rejects subjects without "Estimate"; `@IsNotEmpty()` prevents empty subjects |
| T-44-12 | Information Disclosure | Email to arbitrary recipients | mitigate | `@IsEmail()` validates format; `EstimatePolicy.canUpdate()` ensures only business owner can trigger sends; no cross-business access |
| T-44-13 | Elevation of Privilege | Sending from non-owner | mitigate | `EstimatePolicy.canUpdate(authUser, estimate)` verifies authUser.businessId matches estimate.businessId |
| T-44-14 | Denial of Service | Email send spam | accept | Rate limiting not in scope for v1.8; Resend has built-in rate limits; accepted risk for solo-user product |
</threat_model>
