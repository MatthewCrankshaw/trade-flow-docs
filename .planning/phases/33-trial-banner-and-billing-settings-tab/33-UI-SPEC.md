## Color

All values from the established Trade Flow colour system (carried forward from Phase 32 UI-SPEC):

| Role | Token | OKLCH | Usage in Phase 33 |
|------|-------|-------|-------------------|
| Dominant (60%) | `background` | `oklch(0.99 0 0)` | Settings page canvas (existing) |
| Secondary (30%) | `card` | `oklch(1 0 0)` | SubscriptionStatusCard surface |
| Accent (10%) | `primary` | `oklch(0.31 0.09 245)` | See reserved list below |
| Destructive | `destructive` | `oklch(0.64 0.21 25)` | Not used in Phase 33 (no destructive actions) |

Accent (`primary`) reserved for:
- "Subscribe" CTA button on billing tab when status is inactive (`bg-primary text-primary-foreground`)
- "Manage Billing" CTA button on billing tab (`bg-primary text-primary-foreground`)

Additional semantic colors:
- TrialChip background: `bg-amber-100 text-amber-800` (light mode) -- warm amber signals urgency without alarm. Uses Tailwind amber palette directly (not a custom token). Provides visual distinction from the neutral header without competing with primary accent.
- TrialChip icon: `text-amber-600` -- `Clock` icon tinted to match chip background
- Status badge "Active": `bg-green-100 text-green-800` -- positive confirmation
- Status badge "Trial": `bg-amber-100 text-amber-800` -- matches TrialChip for consistency
- Status badge "Cancels on...": `bg-orange-100 text-orange-800` -- warning state
- Status badge "Inactive": `bg-muted text-muted-foreground` -- neutral/dormant

## Interaction States

### TrialChip

| State | Condition | Visual |
|-------|-----------|--------|
| Hidden | `isLoading === true` | Not rendered (prevents flash) |
| Hidden | Status is NOT `trialing` | Not rendered |
| Hidden | Calculated days remaining is negative | Not rendered (webhook lag edge case, D-07) |
| Visible | Status is `trialing` AND days >= 1 | Amber chip: `Clock` icon + "X days left in trial" (D-05) |
| Visible (last day) | Status is `trialing` AND days === 0 | Amber chip: `Clock` icon + "Last day of trial" (D-06) |
| Hover | Mouse over chip | `cursor-pointer`, subtle hover background darken (`bg-amber-200`) |
| Click | User clicks chip | Navigate to `/settings?tab=billing` (D-04) |

### SubscriptionStatusCard

| State | Condition | Visual |
|-------|-----------|--------|
| Loading | `isLoading === true` (D-16) | Skeleton placeholder for status badge, date line, and CTA button |
| Trialing | `status === 'trialing'` | Badge "Trial", status line "Trial -- X days remaining", CTA "Manage Billing" |
| Active | `status === 'active'` AND `cancelAtPeriodEnd === false` | Badge "Active", status line "Active -- renews [date]", CTA "Manage Billing" |
| Cancelling | `status === 'active'` AND `cancelAtPeriodEnd === true` (D-13) | Badge "Cancels on [date]", status line "Cancels on [date]", CTA "Manage Billing" |
| Inactive | `status` is `past_due` / `canceled` / no subscription | Badge "Inactive", status line "Inactive", CTA "Subscribe" |
| CTA loading | Portal session being created | "Manage Billing" button shows `Loader2` spinner, disabled (D-15) |

### Settings Page Tab Routing

| State | Condition | Visual |
|-------|-----------|--------|
| Default tab | No `?tab=` param or `?tab=general` | Show existing default settings content |
| Billing tab | `?tab=billing` (D-09) | Show BillingTab with SubscriptionStatusCard |
| Portal return | User returns from Stripe Portal | `?tab=billing` preserved in URL, Billing tab pre-selected (D-09) |

### Date Formatting

All dates rendered using the format: `"15 Jan 2026"` (day-month-year, abbreviated month). Consistent with UK locale conventions for a UK-targeted tradesperson app. Implement via `Intl.DateTimeFormat('en-GB', { day: 'numeric', month: 'short', year: 'numeric' })` or equivalent Luxon formatting.

### Error States

| Scenario | Copy |
|----------|------|
| Portal session creation failure | Toast: "Unable to open billing portal. Please try again." |
| Subscription fetch failure | Standard RTK Query error handling (existing pattern) |

### Empty / No Subscription State

| Element | Copy |
|---------|------|
| Status badge | "Inactive" |
| Status description | "You don't have an active subscription." |
| CTA | "Subscribe" (navigates to `/subscribe`) |

### Destructive Actions

None in this phase. Cancellation is handled entirely through the Stripe Billing Portal (D-12). No in-app cancel button.

## Accessibility Notes

- TrialChip is a clickable element: implement as `<a>` or `<button>` with `role="link"` and descriptive `aria-label="X days left in trial, view billing settings"`
- TrialChip uses sufficient color contrast: amber-800 on amber-100 meets WCAG AA for small text
- "Manage Billing" button includes `ExternalLink` icon -- add `aria-label="Manage Billing (opens Stripe)"` or visible "(opens in new tab)" text
- Status badge text is readable by screen readers (not icon-only)
- Skeleton loading states use `aria-hidden="true"` and parent has `aria-busy="true"`
- Tab navigation follows standard keyboard pattern: arrow keys between tabs, Enter/Space to activate
