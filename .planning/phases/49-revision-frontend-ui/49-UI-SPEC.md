## EstimateActionStrip Integration Contract

### Edit and resend button (Sent / Viewed / Responded / SiteVisitRequested status)

Source: RESEARCH.md Pattern 2, requirement REV-02.

| Property | Value |
|----------|-------|
| Label | "Edit and resend" |
| Icon | `FilePenLine` (lucide) `h-4 w-4` |
| Variant | `default` |
| Size | `sm` |
| Position | In EstimateActionStrip, before "Convert to Quote" button |
| Condition | `estimate.status IN ["sent", "viewed", "responded", "site_visit_requested"]` |

### Button states

| State | Button |
|-------|--------|
| Ready | `<Button size="sm"><FilePenLine className="h-4 w-4" /> Edit and resend</Button>` |
| Creating revision | `<Button size="sm" disabled><Loader2 className="h-4 w-4 animate-spin" /> Creating revision...</Button>` |
| Error | Button returns to Ready state |

### No confirmation dialog

Source: RESEARCH.md Open Question 2 recommendation.

The revision creates a Draft that the trader must explicitly send. The "send" step is the real commitment. No confirmation dialog needed. This matches the existing "Convert to Quote" pattern in Phase 47.

## Edit and Resend Interaction Flow

Source: RESEARCH.md Pattern 1 and Pattern 4, requirement REV-02.

### Flow sequence

1. Trader clicks "Edit and resend" button
2. Button text changes to "Creating revision..." with `Loader2 animate-spin` icon, button disabled
3. `POST /v1/estimates/:id/revisions` fires (no request body)
4. On success: `toast.success("New revision created")` then navigate to `/estimates/{newRevisionId}` (the returned estimate's ID)
5. On error: `toast.error("Failed to create revision. Please try again.")`, button re-enabled

### Post-navigation state

The user lands on the new Draft revision's detail page. The estimate number remains the same (E-YYYY-NNN) but `revisionNumber` is incremented. The estimate is in Draft status, ready for editing. The History card now includes the previous revision.

## EstimateRevisionHistory Contract

Source: RESEARCH.md Pattern 3, requirement REV-04.

### Render condition

Only render when `estimate.revisionNumber > 1`. Root estimates with `revisionNumber: 1` have no history to show.

### Data fetching

Eager-fetch on page mount when `revisionNumber > 1` (source: RESEARCH.md Open Question 1 recommendation). Use `skip: estimate.revisionNumber <= 1` on the `getEstimateRevisions` RTK Query hook. This avoids a loading flash when expanding the collapsible.

### Placement

Below the action strip and above line items on the EstimateDetailPage, same level as other detail cards.

### Structure (collapsed -- default state)

```
+------------------------------------------+
| (History icon) History          [v]      |
+------------------------------------------+
```

### Structure (expanded)

```
+------------------------------------------+
| (History icon) History          [^]      |
|                                          |
|  Revision 2 (current)                   |
|  Sent 15 Apr 2026, 10:30               |
|  Viewed 15 Apr 2026, 14:22             |
|                                          |
|  --------------------------------        |
|                                          |
|  Revision 1                              |
|  Sent 12 Apr 2026, 09:15               |
|  Viewed 12 Apr 2026, 11:00             |
+------------------------------------------+
```

### Revision list item layout

Each revision renders as a block within the collapsible content:

| Property | Value |
|----------|-------|
| Container | `flex flex-col gap-1` per revision item |
| Revision label | `text-sm font-semibold text-foreground` -- "Revision {N}" or "Revision {N} (current)" for `isCurrent: true` |
| Current badge | Inline text `(current)` in `text-muted-foreground font-normal` appended to label |
| Sent line | `text-xs text-muted-foreground` -- "Sent {formatDateTime(sentAt)}" or omitted if `sentAt` is null (Draft revision) |
| Viewed line | `text-xs text-muted-foreground` -- "Viewed {formatDateTime(firstViewedAt)}" or omitted if `firstViewedAt` is null |
| Not sent state | `text-xs text-muted-foreground` -- "Draft -- not yet sent" when `sentAt` is null |
| Item separator | `border-b border-border` between items, no border after last item |
| Item padding | `py-3` per item (12px vertical padding) |

### Revision list ordering

Display newest revision first (reverse the API response which returns oldest-first). The current revision appears at the top.

### Loading state

While `getEstimateRevisions` is loading, show a single line of skeleton text inside the collapsible content: `<div className="h-4 w-48 animate-pulse rounded bg-muted" />` repeated twice.

### Error state

If the revisions query fails, render inside the collapsible content: `<p className="text-sm text-muted-foreground">Could not load revision history.</p>`

### Empty state

Not applicable -- the component only renders when `revisionNumber > 1`, so there is always at least one previous revision.

## State Contracts

### Loading State (Edit and resend)

No dialog or skeleton needed. The "Edit and resend" button shows a spinner during the POST. All other page content remains visible and interactive (except the action strip buttons which are part of the same component).

### Loading State (History)

Collapsible content shows skeleton placeholders while the revisions query loads. The card header and toggle remain interactive.

### Error States

| Condition | Display | Copy |
|-----------|---------|------|
| Revision creation failure (server error) | Toast error | `toast.error("Failed to create revision. Please try again.")` |
| Revision creation failure (status invalid) | Toast error | `toast.error("This estimate cannot be revised in its current status.")` |
| Revision creation failure (network) | Toast error | `toast.error("Could not connect to the server. Check your connection and try again.")` |
| Revision history load failure | Inline text in collapsible | "Could not load revision history." |

### Empty States

No empty states in this phase. The "Edit and resend" button only appears on existing estimates, and the History section only renders when `revisionNumber > 1`.
