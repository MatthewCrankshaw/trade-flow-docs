    **Route 2: POST /v1/webhooks/stripe (per WBHK-01, D-10)**
    This route MUST use `@Public()` decorator to bypass JwtAuthGuard. It MUST be on a SEPARATE controller path. Since the class is `@Controller("v1/subscription")`, the webhook route needs a different approach.

    Create a SECOND controller class in the same file (or a separate file if the codebase convention prefers one controller per file -- check quote.controller.ts pattern):

    ```typescript
    @Controller("v1/webhooks")
    export class WebhookController {
      constructor(
        @Inject(STRIPE_CLIENT) private readonly stripe: Stripe,
        private readonly configService: ConfigService,
      ) {}
      private readonly logger = new AppLogger(WebhookController.name);

      @Post("stripe")
      @Public()
      async handleStripeWebhook(
        @Req() req: RawBodyRequest<Request>,
        @Headers("stripe-signature") signature: string,
      ) {
        const webhookSecret = this.configService.get<string>("STRIPE_WEBHOOK_SECRET");
        try {
          const event = this.stripe.webhooks.constructEvent(
            req.rawBody,
            signature,
            webhookSecret,
          );
          // Phase 30 will add event type handlers here
          this.logger.log(`Received Stripe webhook event: ${event.type}`);
          return { received: true };
        } catch (error) {
          this.logger.error(`Webhook signature verification failed: ${error.message}`);
          throw new InvalidRequestError(
            ErrorCodes.INVALID_REQUEST,
            "Webhook signature verification failed",
            error.message,
          );
        }
      }
    }
    ```

    Import `RawBodyRequest` from `@nestjs/common`. Import `Request` from `express`. Import `@Public()` from wherever the existing decorator is defined (check auth.guard.ts or look for a public.decorator.ts file). Import `@Headers` from `@nestjs/common`.

    The `InvalidRequestError` with the existing `ErrorCodes` enum value will map to HTTP 422. Check if there's a more appropriate error code or if a `BadRequestException` from `@nestjs/common` (HTTP 400) is better for signature failures -- the success criteria says "returns 400". If the existing `InvalidRequestError` maps to 422 (not 400), use `throw new BadRequestException("Webhook signature verification failed")` instead to get the correct 400 status code.
