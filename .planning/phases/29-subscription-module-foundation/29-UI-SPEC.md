> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Applicability

**This phase has no frontend work.** Phase 29 is API-only (trade-flow-api repo). It creates:

- NestJS SubscriptionModule with Stripe SDK factory provider
- MongoDB subscription entity, repository, and SubscriptionStatus enum
- `POST /v1/subscription/checkout` endpoint (returns `{ url: string }`)
- `POST /v1/webhooks/stripe` endpoint (Stripe-to-server, no human interaction)
- `rawBody: true` configuration in main.ts

No UI components, pages, screens, or user-facing elements are produced. All frontend subscription work begins in Phase 32 (Subscription Gate and Subscribe Pages) and Phase 33 (Trial Banner and Billing Settings Tab).

**Requirements covered:** ACQ-02, WBHK-01 -- both are API contracts with no visual output.
