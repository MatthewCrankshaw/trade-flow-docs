    **1. SubscriptionStatus enum** (`src/subscription/enums/subscription-status.enum.ts`, per D-09):
    ```typescript
    export enum SubscriptionStatus {
      TRIALING = "trialing",
      ACTIVE = "active",
      PAST_DUE = "past_due",
      CANCELED = "canceled",
      INCOMPLETE = "incomplete",
    }
    ```

    **2. Subscription entity** (`src/subscription/entities/subscription.entity.ts`, per D-07, D-08):
    Follow the `I{Name}Entity` pattern from base.entity.ts. Interface with:
    - `_id: ObjectId`
    - `userId: string`
    - `stripeCustomerId: string`
    - `stripeSubscriptionId?: string` (optional -- populated in Phase 30)
    - `status: string` (SubscriptionStatus enum value)
    - `currentPeriodEnd?: Date`
    - `trialEnd?: Date`
    - `cancelAtPeriodEnd: boolean`
    - `canceledAt?: Date`
    - `createdAt: Date`
    - `updatedAt: Date`

    **3. Subscription DTO** (`src/subscription/data-transfer-objects/subscription.dto.ts`):
    Follow `I{Name}Dto` pattern. Same fields as entity but with `id: string` instead of `_id: ObjectId`.

    **4. Subscription response** (`src/subscription/responses/subscription.response.ts`):
    Follow `I{Name}Response` pattern from business.response.ts. Define `ISubscriptionResponse` interface and `ICheckoutResponse` with `{ url: string }`.

    **5. Stripe factory provider** (`src/subscription/providers/stripe.provider.ts`, per D-12):
    Follow the EmailModule/SendGrid factory provider pattern from email.module.ts:
    ```typescript
    import Stripe from "stripe";
    import { ConfigService } from "@nestjs/config";

    export const STRIPE_CLIENT = "STRIPE_CLIENT";

    export const stripeProvider = {
      provide: STRIPE_CLIENT,
      useFactory: (configService: ConfigService): Stripe => {
        return new Stripe(configService.get<string>("STRIPE_SECRET_KEY"));
      },
      inject: [ConfigService],
    };
    ```

    **6. Subscription repository** (`src/subscription/repositories/subscription.repository.ts`):
    Follow BusinessRepository/QuoteRepository patterns. Use `MongoDbWriter` and `MongoDbFetcher` from CoreModule. Collection name: `"subscriptions"`. Must include:
    - `static readonly COLLECTION = "subscriptions";`
    - `async create(dto: ISubscriptionDto): Promise<ISubscriptionDto>` -- converts DTO to entity, inserts, returns DTO
    - `async findByUserId(userId: string): Promise<ISubscriptionDto | null>` -- finds by userId field
    - `async upsertByUserId(userId: string, dto: Partial<ISubscriptionDto>): Promise<ISubscriptionDto>` -- upserts using userId as the key
    - Entity-to-DTO and DTO-to-entity mapping methods following existing repository patterns
    - Index creation in constructor or via an `ensureIndexes` method: unique index on `{ userId: 1 }` and unique sparse index on `{ stripeSubscriptionId: 1 }`

    **7. SubscriptionModule** (`src/subscription/subscription.module.ts`, per D-12):
    Standard NestJS module. Import nothing extra (CoreModule is global). Providers: `stripeProvider`, `SubscriptionRepository`. Controllers: none yet (Plan 02). Exports: `SubscriptionRepository`, `STRIPE_CLIENT`.

    **8. Register in AppModule** (`src/app.module.ts`):
    Add `SubscriptionModule` to the `imports` array. Add the import statement: `import { SubscriptionModule } from "@subscription/subscription.module";`
