<objective>
Bootstrap Playwright E2E test infrastructure in trade-flow-ui: install packages, configure the test runner with a three-project layout (setup / chromium / chromium-unauth), implement programmatic Firebase auth that saves storageState with IndexedDB support, and deliver two smoke tests validating both the unauthenticated login page render and the authenticated dashboard landing.

Purpose: Phase 24 creates the foundation every subsequent E2E phase depends on. Without working auth storageState reuse, every test would bear the full Firebase sign-in latency. Without correct IndexedDB capture, authenticated tests silently fail because Firebase v12 stores tokens in IndexedDB, not localStorage.

Output: A green `npx playwright test` run with two passing smoke tests and `e2e/.auth/user.json` written to disk after setup.

  <action>
    Create `trade-flow-ui/e2e/helpers/auth.setup.ts` — the setup project's entry point. This file:
    1. Pings the NestJS API health check endpoint (D-07) and aborts with a descriptive error if unreachable
    2. Navigates to the app root and authenticates via `signInWithEmailAndPassword` called inside `page.evaluate` (D-01 — programmatic, not UI click-through)
    3. Waits for the post-auth redirect to `/dashboard`
    4. Saves `storageState` with `indexedDB: true` — CRITICAL for Firebase v12 which stores tokens in IndexedDB, not localStorage (Pitfall 1 avoidance)

    IMPORTANT: Use `signInWithEmailAndPassword` inside `page.evaluate`, NOT `signInWithCustomToken`. Custom tokens are minted by Firebase Admin SDK and are a different format from the ID tokens returned by the REST API (Pitfall 2 avoidance, per RESEARCH.md anti-patterns section).

    ```typescript
    import { test as setup } from "@playwright/test";
    import * as path from "path";

    const AUTH_FILE = path.join(__dirname, "../.auth/user.json");

    setup("authenticate", async ({ page }) => {
      // Step 1: API health check (D-07) — abort early with a clear message if API is not running
      const apiUrl = process.env.E2E_API_URL ?? "http://localhost:3000";
      const pingResponse = await fetch(`${apiUrl}/v1/ping`).catch(() => null);
      if (!pingResponse?.ok) {
        throw new Error(
          `API not reachable at ${apiUrl} — start docker-compose and the NestJS server first.`,
        );
      }

      // Step 2: Navigate to the app root so Firebase SDK is initialised in the browser
      await page.goto("/");

      // Step 3: Authenticate inside the browser context using Firebase SDK (D-01)
      // Uses signInWithEmailAndPassword (not signInWithCustomToken — different token type)
      // Dynamic import works because trade-flow-ui serves firebase as ESM via Vite
      await page.evaluate(
        async ({ email, password, firebaseConfig }) => {
          const { initializeApp, getApps } = await import("firebase/app");
          const { getAuth, signInWithEmailAndPassword } = await import("firebase/auth");
          const app = getApps().length > 0 ? getApps()[0] : initializeApp(firebaseConfig);
          const auth = getAuth(app);
          await signInWithEmailAndPassword(auth, email, password);
        },
        {
          email: process.env.E2E_TEST_USER_EMAIL!,
          password: process.env.E2E_TEST_USER_PASSWORD!,
          firebaseConfig: {
            apiKey: process.env.E2E_FIREBASE_API_KEY,
            authDomain: process.env.E2E_FIREBASE_AUTH_DOMAIN,
            projectId: process.env.E2E_FIREBASE_PROJECT_ID,
          },
        },
      );

      // Step 4: Wait for AuthProvider's onAuthStateChanged to fire and redirect to dashboard
      await page.waitForURL("**/dashboard", { timeout: 15000 });

      // Step 5: Save storageState including IndexedDB — required for Firebase v12
      // Without indexedDB: true, Firebase auth tokens are not captured (Pitfall 1)
      await page.context().storageState({ path: AUTH_FILE, indexedDB: true });
    });
    ```
