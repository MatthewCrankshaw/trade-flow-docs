<objective>
Fix the onboarding flow so default items reference the correct default tax rate, and update quote factories to resolve tax rate percentages from the taxRateId reference at quote creation time.

Purpose: These are the two downstream consumers of the item taxRateId field. Without these changes, onboarding creates items with taxRateId: null (invalid for non-bundles), and quotes would have no tax rate data.

Output: Onboarding creates items with correct tax rate references. Quote factories resolve tax rate percentages from taxRateId. All tests pass.
</objective>
