<task type="auto">
  <name>Task 1: Add CI gate to API Dockerfile</name>
  <files>trade-flow-api/Dockerfile</files>
  <action>
In the trade-flow-api Dockerfile, add `RUN npm run ci` after the `COPY . .` line and before the `RUN npm run build` line in the builder stage.

The builder stage should become:

```dockerfile
FROM node:22-slim AS builder
RUN apt-get update && apt-get install -y --no-install-recommends python3 make g++ && rm -rf /var/lib/apt/lists/*
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run ci
RUN npm run build
RUN npm run build:worker
```

This ensures all quality checks (tests, lint, format, typecheck) pass before the build proceeds. If any check fails, the Docker build fails and Railway rejects the deployment.
  </action>

<task type="auto">
  <name>Task 2: Create nixpacks.toml for UI CI gate</name>
  <files>trade-flow-ui/nixpacks.toml</files>
  <action>
Create a new file `nixpacks.toml` in the trade-flow-ui root directory. This file configures Railway's Nixpacks builder to run the CI quality gate before building.

Contents:

```toml
[phases.build]
cmds = ["npm run ci", "npm run build"]
```

This overrides the default Nixpacks build phase to run `npm run ci` (tests + lint + format + typecheck) before `npm run build`. If any quality check fails, the build fails and Railway rejects the deployment.

Note: Nixpacks automatically handles the install phase (`npm ci`) so we only need to override the build phase.
  </action>
