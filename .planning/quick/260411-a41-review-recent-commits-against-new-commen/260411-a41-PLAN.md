## Scope

Older commits touched by these same changes are out of scope — their comments are

commits".

## Standard being applied

Per `CLAUDE.md`:

- Prefer self-documenting code over comments. If a comment feels necessary, first
  consider renaming or extracting a method/function/component.
- Acceptable comments: legal headers, `TODO` with ticket numbers, warnings about
  non-obvious consequences, clarification of intent when no better naming exists.
- Never commit commented-out code — use version control history instead.
- Never write comments that restate what the code does.
