<objective>
Fix Resend email delivery errors returning generic 500 Internal Server Error instead of proper 502 Bad Gateway with structured error body.

Purpose: When Resend rejects an email (e.g., "You can only send testing emails to your own email address" in dev), the API currently returns a generic 500 because `createHttpError` in the controller catch block does not recognize `HttpException` instances -- it falls through to the default unknown error handler. The `EmailSenderService` already throws `HttpException` with `BAD_GATEWAY`, and `QuoteEmailSender.send()` re-throws it, but the controller's `createHttpError` wraps it as 500.

Output: Structured 502 responses for email delivery failures with meaningful error codes, messages, and details.
</objective>
